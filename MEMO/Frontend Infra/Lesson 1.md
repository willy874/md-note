**# Lesson 1**

**# What is Fronted Infrastructure?**

今天，先來聊聊前端基礎建設這個主題之前，我想先跟大家分享一個小故事。

小竹是一位前端新手，他剛加入了一家公司。他擁有的技能包括HTML、CSS和 jQuery，這些技能足以讓他應對大多數業務需求，他能夠成功地完成工作任務。

有一天，小竹的主管交給了他一個新專案，這是一個使用框架進行開發的專案。一開始，他對學習框架開發感到很高興，第一次體驗到了框架開發的樂趣，並深刻感受到框架開發帶來的好處。 由於這種開發經驗非常有趣，他決定自己建立一個小專案來嘗試一下，但他很快就發現，要創建這樣的專案並不容易。因為框架需要很多不是原生 HTML 的語法，而且越寫越多，CSS, JavaScript, Images... 等等檔案弄的很亂。

於是，小竹開始學習像Webpack這樣的工具，了解打包、編譯和預處理是什麼。但是這些配置似乎太多太複雜了，他不斷看著終端輸出各種紅色的錯誤訊息，他感到非常沮喪。直到他遇到了專案建置工具，他才體會到了便利之處。 市面上常見的工具，如Vue-Cli、Create React App 等，大大提高了開發效率和速度，節省前期配置這些設定。

儘管一開始使用這些工具讓小竹充滿成就感，覺得自己也可以像主管一樣建構專案了。然而，成就感才剛湧上心頭，下一個挑戰就來臨了。他開始寫程式碼，立刻遇到了Linter 錯誤，終端機一直報各種錯誤，像是缺少空格、多餘的括號、缺分號、程式碼太長等等。 他只能眼睛幾乎貼在螢幕上，逐行找出自己的錯誤在哪裡。 直到他發現原來VSCode有插件可以直接提示他哪裡出錯，他才能快速找到問題所在。

在經歷了ESLint的折磨後，雖然程式碼的可讀性提高了，但編寫速度卻大幅下降。了更有效率地編寫程式碼，小竹找到了自動格式化工具Prettier。它可以與ESLint 規則進行集成整合，只需一次保存操作就能瞬間解決格式問題。 這不僅提供了更好的程式碼閱讀體驗，還節省了整理格式的時間，並顯著提高了開發品質和效率。

儘管在前端工程的成長道路上，許多經驗豐富的前輩告訴小竹「前端沒有什麼困難的，只是做一些介面和效果，將數據呈現在介面上」。 他開始對自己產生懷疑，是否應該朝著全端工程師的方向發展，學習有關資料庫、其他程式語言等不同的技術領域。 然後有人告訴他，你可以學習「Node.js」哦！ 原來前端也可以使用JavaScript來操作伺服器和資料庫，他開始意識到「前端」和「後端」的分界並沒有那麼重要，真正關鍵的是理解這個網路軟體世界的運作方式。

隨著小竹的成長，他開始面臨自己建立專案的挑戰，並對各種開發需求提出了更多的要求。 這些專案建構工具提供的基本功能開始顯得不夠用了。 然後，他又面臨了諸如CI/CD、套件管理、倉庫管理等問題，他深切感受到了「前端」可以如此複雜。

這個小竹的故事基本上就是我身為前端工程師的成長歷程，也是我在專案建構上所經歷的心路歷程。 直到最近，人們開始定義這一切，我們稱之為「前端基礎建設」（Frontend Infrastructure）。

前端基礎設施（Frontend Infrastructure）是指支援和促進前端開發工作的技術和工具的集合。 前端開發涉及創建和維護 UI 介面，這是用戶與應用程式或網站互動的部分。 前端基礎架構宗旨在簡化前端開發流程、提高效能、可維護性和可擴展性，並確保使用者體驗良好。

首先我們可以定義一下 「Frontend Infrastructure 」的範疇到底包含哪些東西？

打包工具：前端建置工具如Webpack、Parcel、Rollup等，用於將多個原始檔案（如HTML、CSS、JavaScript、映像等）打包成最佳化的靜態資源。 它們還可以執行各種任務，如程式碼壓縮、模組化管理、熱重載等。

自動化測試工具：這些工具用於自動化測試前端應用程序，以確保程式碼的品質和穩定性。 常見的前端測試工具包括Jest、Mocha、Cypress等。

程式碼品質工具：靜態程式碼分析工具如ESLint和Stylelint用於檢查程式碼風格和潛在錯誤，以確保一致性和可維護性。

版本控制系統：如Git，用於追蹤和管理程式碼更改，也可以追加規則讓 Git 更加嚴謹。也有像是 ChangeLog 這樣的工具控管版本迭代，以便團隊協作和版本控制。

套件管理器：像npm、yarn、pnpm這樣的工具用於管理和共享依賴項，使前端專案的依賴關係更加可管理。

效能優化工具：工具如Lighthouse和PageSpeed Insights幫助評估和改善前端效能，包括頁面載入時間、資源最佳化等。

前端框架和函式庫：前端開發經常依賴各種框架和函式庫，如React、Angular、Vue.js等，它們提供了建構使用者介面所需的元件和功能。也可以建構各種符合團隊需求專用的前端 UI 元件與函式。

持續整合/持續交付（CI/CD）工具：用於自動化建置、測試和部署前端應用程式的工具，確保程式碼更快地交付到生產環境。

效能監控和錯誤追蹤工具：這些工具用於監視前端應用程式的效能和捕獲用戶報告的錯誤，以及幫助開發團隊及時修復問題。

本地開發伺服器：用於在開發過程中快速查看和測試應用程式的工具，通常支援熱重載，使開發過程更有效率。

前端基礎設施的目標是提高前端開發的效率，確保應用程式的效能和質量，並支援團隊協作。 這些工具和流程有助於前端開發人員更好地管理專案，提供更好的開發者體驗，同時降低維護成本。
