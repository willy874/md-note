**# Lesson 2**

**# Build the Project**

既然要談基礎建構，那我們重頭來看專案的建構這件事。

建置專案第一步你會想做什麼？當然是確認啟動會動就好。

早期是地獄 Webpack，光是要設定好一個專案能順利運行就是多可怕的事情...

到現今都使用 Vite 的時代，簡單選一下選項建出專案雛形，然後從舊專案把常用的設定檔複製過來，那就當是完成了，緊接著就如火如荼裝起常用的套件開始開發業務需求。

但現實肯定不是那麼順利，一下子是 ESLint 噴錯，一下子是 Prettier 不會動，一下是套件有衝突，怎麼會那麼累，還不知道發生什麼事？！接下來的篇章會一篇篇教導你如何從零搭建一個有自己風格的專案，逐步完成這龐大的旅程。

這些 Builder Tool 這麼複雜，但該怎麼學習？

首先要先思考新專案的專案性質來決定要使用的工具，不要任何情況都直接衝動使用同一組工具和解決方案去開發，要去熟悉不同工具他們的特色。

第一，確認你要運行的 Nodejs 版本，前端工具生態鏈一切基石就是 Nodejs。甚致會影響接下來使用的工具和，所以如果已經有預計的作業系統、開發工具、部署環境等等，都得確認對於 Nodejs 的版本支援狀況。甚至會需要定時去升級版本確保系統的穩定和漏洞的補足。

第二，確認你使用的 Project Builder 工具是否符合你的工作情境，雖然現在大多的應用場域都會推薦你使用 Vite，它不會為特定框架服務，它支援最多的框架。然而你仍然要思考，你使用的框架是否支援，比如說 Angular 生態系仍然要使用對應生態系的工具。或許你會說使用各種插件仍然能在 Vite 開發 Angular ，但依然是得考慮工具的成熟度和穩定性。

再來就是開發環境，並不是每個人開發環境都可以給你選擇，你要先了解公司的開發環境是不是能支援 Vite 的開發引擎。舉例來說，Vite 是使用 ESModule 進行開發，如果公司是不能任意使用新版本瀏覽器，那你就得關注版本上的支援狀況。

第三，就是語法轉譯，最常見的語法轉譯問題就是最新的ES 最新的版本使用的語法、TypeScript 語法、語法糖(JSX)、特殊檔案解析(.vue, .mdx)⋯等等，使用工具最常見的就是 babel，支援最穩定，插件涵蓋情境最廣，下載量最多。但相對的缺點就是比較舊，所以插件設定複雜、學習曲線高、轉譯速度也不夠快。當然也可以使用像是 esbuild, swc, rolldown... 之類的新工具處理，但就得面臨更多的挑戰。

第四，就是許多人的天堂與惡夢，ESLint & Prettier 。天堂是讓你的語法與團隊統一，並且自動排版不需要手動調整。惡夢也是，因為套件系版更速度極快，整合套件多，設定複雜，意外情境多。設定這個真的是新手殺手，橫屍片野，但卻又非常需要。習慣使用 Prettier 的人要脫離真的會很想死，排版真的是超級超級花時間。當然慢慢也有新的工具 biome, oxlint/oxfmt ... 等等。

第五，就是要決定框架的周邊技術，接著遇上的問題就有 CSS 解決方案的選用、狀態管理器的選用、UI 元件的選用......等等工具。這就是一個很大的議題了，會有各種的工具，各自的評估與考量。

如果從 Webpack 開始學就是要逐步設定各種東西，但實在是很容易挫折感很重。我還是會建議，如果要入門開始，首先先能夠用 Vite 來啟動一個最簡單的專案，來看看有哪些能夠調整的東西吧！
