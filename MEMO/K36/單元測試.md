# 單元測試

經過幾個需求，我有嘗試撰寫一些單元測試和E2E，目前執行上的困難點。

測試在業務層導入之可行性：

- 有關問題

1. 沒有足夠的 Testid 節點可以設計腳本
2. Field 的更新有 debounce 的延遲不容易模擬
3. 許多 nrc 元件沒有開放 node 參數，導致難以用 testid 進行測試
4. 部分 package 未能夠正確支援 CommonJS (ex: @dnd-kit)
5. I18n 實作不易測試，也不易隔離
6. API 無法預期欄位、屬性、邏輯，沒有相關完整文件可以確認進行模擬
7. 大量靠 Webpack 注入參數，不易模擬
8. 專案 context, store 複雜，不易模擬
9. 登入人數限制一人，難以併發測試

- 基於 2, 3 這幾點是新頁面寫測試的關鍵原因
- 基於 1, 2, 3, 6 是舊頁面很難補上測試的原因
- 基於 1, 3, 9 是 E2E 測試難以撰寫的原因

目前基礎上能做的：

- 開始要逐步補上 testid 的習慣
- 元件進行拓展，要在對應功能開出 Node 相關的參數
- 元件進行重構，重新設計 form 的資料流，FormValues 不應該是非同步更新
- 針對做到的需求慢慢補上型別文件(目前新需求我這邊都會補上)
