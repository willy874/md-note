# Shadcn UI

近期覺得 Shadcn UI 氾濫，切版用、入門用、後台用、AI也推薦你用，看到 PoC 用、學生用、Side Project 用、面試者用，現在幾乎成為前端的優先選型，老實說有點疲乏。

我同意 Headless 系列的 UI Library 帶來極強的可修改性，但我覺得相對的也帶來很大的學習與閱讀負擔。甚至必須說很多細節的功能要你自己去實現。

沒有一套「完美的 Library 」，工程師永遠在「彈性」和「效率」之間取捨。選了彈性會犧牲點效率，選了效率會犧牲點彈性，這通常都是二擇一。做了幾年的工程師，選技術考量的已經不是「喜好」和「習慣」，而是「取捨」和「合適」。
