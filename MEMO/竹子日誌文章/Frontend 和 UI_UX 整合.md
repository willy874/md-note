# Frontend 和 UI/UX 整合

## 產品設計的碎碎念 ep03

既然開始寫 Spec，也要開始畫 UI。工程師背景的我就想要找找有沒有 Open Source 的資源，也想找各種整合資源。

原本在想 Material Design 和 MUI 本身就是高度整合的組合，理所當然認為有豐富的開源資源可以用，可以快速幫助開發。身為工程師背景的我，原本想要用熟悉開發生態這件事，用最少量的設計去快速完成基礎的介面。

但似乎沒那麼簡單⋯⋯

因為實際上在「UI 設計」的領域中，並沒有那麼多免費資源，資源與前端的整合度和同步速度有非常大的落差，在整個設計資源中對於開源這個角度並沒有像是工程師圈這麼蓬勃。所以我大概能理解「造輪子」的現象似乎有點普遍，因為免費資源的匱乏和與工程對齊等等都有頗大的誤差，最後不免要在設計稿和工程師所使用的 UI 兩者不斷對校。

雖然 Figma 已經努力用各種方法消弭「開發」和「設計」的距離，但真要說對齊恐怕還有一段路。實作過程中還是各種綁手綁腳的感覺，也或許是我帶入太多開發思維，也或許是我還沒找到方法⋯⋯
