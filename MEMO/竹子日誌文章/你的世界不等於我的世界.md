# 你的

# 世界

# 不等於

# 我的

# 世界

#

日前我還在找工作時跟一個老工程師的公司主管面試，真的是越聊越生氣。只能說每個人認知的世界不太一樣，不需要把自己世界的認知強加到別人身上。

原本問到我的生涯目標，我表示我是以「管理職的規劃在找工作」。被質疑我都是做前端，想上管理職必須要「做後端」才行。其實也沒實際深聊過任何後端技術，只是看看經歷就拋一句「前端還是要熟悉後端才能當主管」。喔！你當我下班後都是在發酵嗎？

但我覺得管理職未必要懂全部的技術，

你要一個後端背景的主管下去畫Figma設計稿可以嗎？

你要一個維運背景的主管下去規劃前端系統架構可以嗎？

你要一個設計背景的主管下去伺服器裡面部署應用程式可以嗎？

我自稱前端工程師並不代表我不懂後端，只是我會很自豪的說「我都很會」，術業有專攻，擅長的領域中就有差。跟我談特定的框架、語言、資料庫是不是太狹隘了？

雖然時隔一段時間，但遇到這種自以為是的老人回想起來還頗不爽。
