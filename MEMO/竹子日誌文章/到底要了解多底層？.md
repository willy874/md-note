# 到底要

# 了解

# 多底層？

#

資深的總是會告訴你，「要多了解底層」。

但你不覺得這個「底層」到底是要多「底層」？

專案架構了解完要看框架函式庫原始碼。

原始碼看完要看協定實作。

協定看完要看組合語言記憶體操作。

組合語言看完要看⋯⋯⋯

何必呢？

我的想法

其實只要穩定且被統一的東西

都不需要了解實作細節

你需要了解的是「要做什麼」、「規範」、「特性」

不是操作介面

不是實現實作

更不是工具

追求的是「本質」

版本迭代，介面和實作會改變

趨勢改變，工具會改變

這些學習逐漸不管用

現在的技術更迭和變化快到不可思議

不斷去追求「底層」其實非常有限

花了很多時間卻沒有效益
