# 工作隨記

開發原則

1. 在業務功能減少使用過多冷門原生API，盡量拉去 utils處理
2. Util 要經過單元測試做確認，功能要夠 pure，最多去依賴非業務常數，詳述輸入輸出
3. 減少使用的方法，能夠羅列業務功能使用到的API
4. 多採用 OOP或 FP 方法來管理資料
5. 依賴系一定要清晰，避免循環依賴
6. 單元測試選用
7. 樣式注入
8. 製作SVG組件，動態抓取圖片字串，並曝露Promise來抓取圖片載入結束的時間點。
9. 非同步組件製作，設計一個好用的區域加載組件。
10. 需要 Observer API polyfill
11. JSX 引用
12. 為什麼要用 reload?
13. 登入機制
14. 快取機制
15. 命名規範設定
16. Kebab Case // class, url
17. Snake Case // table column, component
18. Screaming Case // enum, constant
19. Camel Case // variable
20. Pascal Case // module, class function
21. Hungarian notation // not use
