# 二階思考（Second-Order Thinking）

## 核心概念

Howard Marks（Oaktree Capital 創辦人）在《The Most Important Thing》中系統化這個概念：

- **一階思考**：這個決定會造成什麼直接結果？
- **二階思考**：然後呢？各方會怎麼反應？
- **三階、四階**：再然後呢？這些反應會帶來什麼連鎖？

Marks 的核心洞察是——**在市場上，一階共識已經 priced in**。如果「這家公司好，所以股價會漲」是人人看得到的一階邏輯，那它早就漲過了；真正的超額報酬來自**看到別人還沒看到的二三階後果**。

這個原則不限於投資。政策、組織規則、產品功能、人際互動——所有**有反饋機制**的系統，一階最佳解常常是錯的，因為沒考慮其他角色會如何應對。

「聰明人看一步，賢者看三步」。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- **政策、策略、組織規則**制定——顯而易見的解往往是錯的
- 使用者只看到**直接後果**，沒想到「然後呢？再然後呢？」
- **投資 / 市場預測**：多數人的一階共識已經反映在價格裡
- **產品功能討論**：加了會誘發什麼用戶行為？競爭對手會怎麼反應？
- 組織決策：加薪、裁員、發新規則——團隊會如何因應？
- **個人關係**：某個話一出口會引發什麼情緒、接下來怎麼收？

不適合時機：

- 日常瑣事、低風險決策——過度推演會癱瘓
- 完全**封閉、無反饋**的決策（純數學題、物理計算）
- 時間極短必須立刻反應——二階思考需要時間

## 如何應用

1. **寫下一階效應**：這個決定的直接結果。
2. **列出會受影響的角色**：用戶、員工、競爭對手、市場、家人⋯⋯
3. **追問「然後呢？」**：每個角色會如何反應？他們的反應又會引發什麼？
4. **推到三階**：多數 surprise 出現在三四階。一階符合預期、二階偶有反撲、三階常常反噬原始意圖。
5. **找出非預期後果**：哪些結果是你**原本沒打算要的**？哪些是反撲機制？
6. **調整決策或加 safeguards**：要嘛改方案、要嘛加預警機制、要嘛接受這些副作用。

## 案例

- **最低工資調漲**：一階 = 低薪工人收入增加。二階 = 部分企業減少雇用或加速自動化。三階 = 剛入職場的新鮮人機會減少。政策好壞取決於淨效應，不是只看一階。
- **高速公路拓寬**：一階 = 塞車減少。二階 = 更多人決定開車上班（誘發需求）。三階 = 幾年後塞回原樣甚至更塞（Induced demand）。
- **無限休假制**：一階 = 員工有彈性、公司顯得福利好。二階 = 沒有明確額度 → 沒人敢請 → 實際休假反而變少。
- **產品加功能**：一階 = 重度用戶開心。二階 = 介面變複雜 → 新用戶流失。三階 = 留存短期上升、長期下降。
- **公司漲薪幅大**：一階 = 士氣高。二階 = 明年期望墊高，漲少了會失望。三階 = 談薪文化被訓練成「漲幅只能更高」。
- **Netflix 砍爛節目**：一階 = 觀眾抱怨。二階 = 創作者更謹慎、留下的節目品質提升。三階 = 品牌從「什麼都有」變成「有品味」。

## 常見誤用

- **只推到二階就停**：二階多半還在「可預期」範圍，真正的 surprise 往往在三四階。
- **把二階思考當精準預測**：它是**掃可能性**，不是算命。目的是看到潛在反撲，不是確定未來。
- **過度應用**：為每個小決定都推到五階——癱瘓。二階思考是**高影響、有反饋**決策的工具。
- **只推自己喜歡的方向**：刻意迴避自己決策會帶來的負面二階——變成合理化而非分析。
- **忽略時間尺度**：某些二階效應需要幾年才會顯現，短期看起來一階解很完美。
- **忽略二階是動態的**：對手 / 用戶 / 員工的反應會隨時間變化，一次二階推演不等於一勞永逸。

## 與其他模型的關係

- **+ [系統思考](../analysis/systems-thinking.md)**（[analysis 模組](../analysis/index.md)）：系統思考看**回饋循環、延遲、槓桿點**，二階思考是其起點——你開始問「然後呢」，就是在進入系統思維。
- **+ [Pre-mortem](pre-mortem.md)**：Pre-mortem 往失敗方向追原因，二階思考是全方向推後果。兩者互補。
- **+ [期望值](expected-value.md)**：二階後果也要納入 payoff 計算——只看一階的期望值常常是錯的。
- **+ [bias 模組](../bias/index.md)**：「這次不一樣」的線性思維、[可得性偏誤](../bias/availability-heuristic.md)（只想到最近的例子）都會阻礙二階思考。
- **+ [反向思考](inversion.md)**：反向思考 + 二階思考 = 「這個決定最壞會引發什麼二三階災難」——重大決策的雙保險。
