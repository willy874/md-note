# Steelman（鋼人論證）

## 核心概念

**討論時，先把對方的觀點重述成它最強的版本，再回應。** 與「稻草人謬誤」（straw man，把對方觀點弱化再打）正好相反——不找對方話裡最好打的那塊，而是**主動幫對方把漏洞補好**，再對最強版本提出反駁。

為什麼要這麼麻煩？三個理由：
1. **智識誠實**：打稻草人贏了也沒贏，只是對一個不存在的論點獲勝。
2. **真正改變對方想法的前提**：對方只要覺得你沒理解他，就不會聽。Steelman 是「我真的懂你在說什麼」的證明。
3. **自己的觀點被壓力測試**：能擊敗最強版的對方，自己的觀點才真的站得住腳。

Anatol Rapoport 的「論戰四規則」第一條：**先用自己的話把對方觀點重述到對方說「對，這就是我的意思，甚至比我講得更好」，才取得反駁資格。**

核心不在「嘴上客氣」，而在**實質理解對方到他自己都點頭**。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者和對方**意見衝突**，但發現自己一直在反駁「自己想像中的對方」而不是對方真正的觀點
- **跨部門 / 跨角色爭論**（工程 vs 產品 vs 設計）：你覺得對方「為什麼要這樣搞」，但對方有你沒看到的限制
- **辯論 / 公開討論**，想培養實質影響力（而非只是嗆贏）
- **家庭 / 伴侶討論**：情緒化你來我往、卻沒真的在聽對方
- 讀到某個立場「覺得根本沒道理」時——這正是該 Steelman 的警訊

不適合時機：
- 對方是**明顯惡意 / 詐欺 / 騷擾** → 不需要 Steelman，需要退出
- **操作型小爭執**（選哪家餐廳）→ 抬槓沒意思

## 如何應用

1. **暫停反駁衝動**：聽完對方，給自己 30 秒「不急著回」。
2. **用自己的話重述對方立場**，盡可能補強：
   - 「所以你的意思是⋯⋯因為⋯⋯對嗎？」
   - 補上對方**沒講但合理**的前提（「而且你應該也考慮到 X，對吧？」）
3. **確認對方點頭**：直到對方說「對，你講得其實比我還清楚」。
4. **然後才進入回應**：針對這個強化後的版本提出你的差異點——而不是「可是你忽略了⋯⋯」（往往對方沒忽略，是你沒聽到）。
5. **公開書寫 / 辯論時**：反駁前插入一段「我先重述對方最強版本」，讀者會立刻覺得你的反駁更可信。

## 案例

- **跨部門會議**：業務抱怨「工程為什麼不快一點上這個功能」。工程不是反駁，而是 Steelman：「我懂你壓力——這個客戶半年內第三次催，合約可能掉；你需要一個能對客戶交代的時間點，對嗎？」業務一旦覺得被理解，對話才進得了下一層（「但工程端有技術債風險⋯⋯」）。
- **政策 / 意識形態爭論**：遇到立場和自己不同的文章，強迫自己先寫「這篇文章最有說服力的那一面是⋯⋯」，再寫反駁。你會發現很多原本「沒道理」的立場其實有你沒看到的脈絡。
- **伴侶爭吵**：「你覺得我不尊重你時間」→ 先 Steelman：「我遲到又沒講讓你感覺被放在比較不重要的位置——而且這不是第一次，累積起來才是重點，對嗎？」情緒會立刻降一半。
- **讀哲學 / 經典**：Daniel Dennett 建議讀任何立場時，先 Steelman 再批判——你學到的會比直接反駁多十倍。

## 常見誤用

- **做表面功夫**：「我懂你的意思啦,但⋯⋯」——其實根本沒 Steelman,只是禮貌性過渡。對方感覺得出來。
- **Steelman 成「同意對方」**：Steelman 不等於認同,是「實質理解」。重述完之後你仍然可以(且應該)提出差異。
- **Steelman 假議題**：對方的論點根本不成立(邏輯錯、事實錯)卻被迫強化——這反而扭曲討論。Steelman 適用於「有合理基礎但結論不同」的觀點。
- **用 Steelman 拖延決策**：「我再想想對方的立場」——無限 Steelman 變成不決策的藉口。重述一次就夠了,接著要推進。

## 與其他模型的關係

- **+ [bias 模組](../bias/index.md)（[確認偏誤](../bias/confirmation-bias.md)、動機推理）**：Steelman 是對抗確認偏誤的直接工具——強迫自己找對方論點的強處,而不是只挑弱處打。
- **+ [受眾思維](audience-first.md)**：Steelman 的本質是「完全站到對方視角」——這是受眾思維的極致版本。
- **+ [第一性原理](../analysis/first-principles.md)**([analysis 模組](../analysis/index.md))：兩者合用——Steelman 理解對方的論證結構,再用第一性原理檢視各自的底層假設分歧在哪。
- **vs [費曼](feynman.md)**：費曼是「能不能講給別人聽」(自己懂不懂),Steelman 是「能不能講對方的立場給對方聽」(懂不懂別人)。都是「重述」的練習,但方向相反。
