# 鄧寧-克魯格效應（Dunning–Kruger Effect）

## 核心概念

由 David Dunning 和 Justin Kruger 在 1999 年的研究提出：**一個人的能力越不足，反而越容易高估自己；能力越好，反而傾向低估自己**。

原因是「評估能力」本身就是一種能力——

- **新手**沒有足夠的知識去知道「自己不知道什麼」（unknown unknowns）。所以他們看不見自己的盲點，誤以為自己差不多懂了。
- **專家**見過這個領域的深廣，知道有多少事自己還沒掌握；同時他們**高估別人也懂**，以為「這很基本吧」。

典型的能力 / 自評曲線：

```
 自評能力
    ↑
高   │       ╱╲         ← 「愚笨山」（Mount Stupid）：新手剛學一點就爆棚
    │      ╱  ╲
    │     ╱    ╲_________________ ← 「絕望谷」：意識到水有多深
中   │    ╱                      ╲
    │   ╱                         ╲_________  ← 「熟練原」：逐步校準
    │  ╱                                    ╲
低   │ ╱                                     ╲←  「大師坡」：專家反而謙虛
    └────────────────────────────→ 實際能力
       入門          中級           專家
```

（坐標為示意，實際研究不完全是這個曲線，但方向正確。）

核心洞見：**信心和能力在低端是脫鉤的**。社群、媒體、職場會議中最敢發言的，常常不是最懂的。這不是個性問題，是認知結構。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者**剛入門某領域**就開始大膽判斷、給他人建議、否定專家
- 或相反：使用者在**自己專長**裡反而覺得「我其實不懂」，不敢發言（專家謙虛過頭）
- 團隊 / 社群中**新人最敢拍板、老手最猶豫**，決策品質與發言量呈反比
- 招募、升遷時看「表達自信度」卻忽略實質能力——可能選到 Mount Stupid 的人
- 使用者遇到「學了三個月就要教別人」的衝動

不適合時機：

- 使用者**有合理的中度自信**——不是每個自信都是偏誤，有些是校準過的
- 討論的是**非能力型決策**（例如偏好、倫理）——Dunning-Kruger 主要講「客觀可評估」的能力

## 如何應用

1. **辨認自己在曲線上哪段**：
   - 剛學不久但講得頭頭是道 → 可能在 Mount Stupid
   - 學了很久反而更不敢斷言 → 可能是專家謙虛，不是「還不懂」
   - 分辨方法：這個領域**有多少種子問題你還沒遇過**？遇過的只是百分之幾？
2. **主動尋求專業回饋**：新手自評不可信，要靠**外部客觀測量**（成績、評分、同儕評審、比賽）來校準。
3. **建立「已知未知」清單**：明確列出「我知道自己還不懂的東西」。這份清單越長，你對自己的校準越好。清單很短的人通常在 Mount Stupid。
4. **專家端：逼自己發言**：專家常因「怕說不準」沉默。但**沉默讓 Mount Stupid 的聲音主導場面**。專家應該表達時帶上信心區間（「我在這個 sub-domain 有 80% 把握，但另一個面向我只有 40%」）而不是全然保留。
5. **對「超強自信」打折扣**：聽到某人毫無猶豫地下大結論時，降權處理。真正的專家在邊界情境多數是「看情況」。
6. **教學是最好的校準器**：試著教別人，你會立刻發現自己哪裡不懂。[費曼技巧](../learning/feynman.md)（[learning 模組](../learning/index.md)）本質上就是 Dunning-Kruger 的體檢表。

## 案例

- **Dunning-Kruger 原始實驗**：測驗文法、邏輯、幽默感後讓受試者自評。最差的 25% 自以為在前 40%；最好的 25% 自以為在前 30%（反而低估）。
- **Reddit / 社群辯論**：學了三個月投資 / 哲學 / 心理學的人最敢對專家下指導棋。看起來像自信，其實是沒意識到自己在 Mount Stupid。
- **醫療 Google 自診**：讀了兩篇文章就覺得診斷得出疾病並質疑醫生。醫生常被這種「半桶水病人」拖累診療流程。
- **工程師 1 年 vs 10 年**：剛寫一年的工程師敢大改架構、否定前人；十年的反而會說「這段我要研究一下再回你」——不是不懂，是見過太多邊界 case。
- **招募面試**：高自信 + 流暢表達的候選人常被高估。有經驗的面試官會**刻意問邊界情境 / 尷尬問題**，專家會說「這個我沒處理過，但我會這樣查」，Mount Stupid 的人會即興編一套聽起來很對的答案。
- **新創 founder**：產業外的新進者常帶著「用科技顛覆 X 產業」的自信，低估了 X 產業幾十年來累積的隱性知識。有的能成功（外部視角有用），很多失敗（Mount Stupid）。

## 常見誤用

- **把 Dunning-Kruger 當貶低別人的工具**：「你有 Dunning-Kruger 啦」變成嘴砲武器，結果自己反而陷在 Mount Stupid（因為你不再接受反駁）。
- **以為「我讀了 Dunning-Kruger 就不會中」**：讀懂偏誤 ≠ 避開偏誤。你讀完這篇仍然會在某領域是 Mount Stupid。
- **變成什麼都不敢講的專家**：過度內化「專家該謙虛」→ 在該發言時沉默 → 讓爛決策通過。謙虛是對**邊界**的，不是對**核心專長**的。
- **用謙虛偽裝無知**：表面說「我不懂啦」其實是不想被挑戰。真正的謙虛會同時展現「我已知的紮實部分」。
- **把能力曲線當固定宿命**：Dunning-Kruger 不是詛咒，是階段。持續練習 + 被打臉 + 被校準，都會讓你往右移。

## 與其他模型的關係

- **+ [過度自信](overconfidence.md)**：Dunning-Kruger 是過度自信的**特殊版本**——專門發生在「能力低端」。過度自信更廣，涵蓋所有能力段。
- **+ [確認偏誤](confirmation-bias.md)**：Mount Stupid 的人不會去找反例，因此偏誤自我強化。
- **+ [倖存者偏誤](survivorship-bias.md)**：新手讀了一本暢銷書就覺得自己「懂這行了」，完全忽略倖存者故事的侷限。
- **+ [第一性原理](../analysis/first-principles.md)**（[analysis 模組](../analysis/index.md)）：新手用第一性原理衝擊專家意見**有時** 反而能帶來突破（馬斯克效應），但更常是 Mount Stupid。差別在於：有沒有真的拆到事實層，或只是無視領域知識。
- **+ [learning 模組](../learning/index.md)的[費曼技巧](../learning/feynman.md)**：教學 / 說給外行聽的壓力，會立刻暴露 Mount Stupid。
- **+ [decision 模組](../decision/index.md)的信心區間 / 校準**：真正的解藥是長期記錄「我的預測命中率」，而不是自我感覺。
