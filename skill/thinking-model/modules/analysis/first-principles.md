# 第一性原理（First Principles Thinking）

## 核心概念

第一性原理源自 Aristotle：「第一性原理是那些不能被進一步拆解、也不能從其他命題推導出來的基本命題。」現代由 Elon Musk 大量引用後廣為人知。

核心假設：**人類多數推理是「類比推理」（analogical reasoning）——因為 X 跟 Y 長得像、以前這樣做過、大家都這樣做，所以我也這樣做。** 類比快但會**繼承過時的假設**。當環境條件改變，類比出來的結論會錯得非常離譜。

第一性原理的做法相反：把問題**拆回不可再拆的基本事實**（物理限制、數學關係、人的基本需求⋯⋯），丟掉所有「大家都這樣」，然後從底下重新往上推演出結論。

這個方法**成本極高**：很慢、很費腦、常常挑戰既有共識，會讓人不耐煩。所以它不是日常工具，而是**關鍵決策的重啟鍵**——當你懷疑自己的前提已經腐爛時才拿出來用。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者的推論大量倚賴**類比、慣例、前例**（「因為以前⋯⋯」「業界都這樣⋯⋯」「OOO 已經做過了」）
- 想**重新定義問題**：創新設計、產業重構、商業模式顛覆
- 既有方案**成本 / 效能撞到天花板**，常識告訴你「再改也就這樣了」
- 要挑戰「不可能」的斷言——先拆清楚「不可能」背後是物理限制還是心理慣性

不適合時機：

- 日常執行性任務——為每件事都從頭推演會癱瘓（Jeff Bezos 講的「disagree and commit」就是提醒不要濫用）
- 已經有成熟最佳實踐的領域（會計、安全、稅務）——前人已經踩過坑，類比通常是對的
- 只是想跟既有方案不同而刻意唱反調——那是反骨，不是第一性原理

## 如何應用

1. **寫下你要挑戰的結論**：例如「我們這類公司不可能做硬體」「這個成本壓不下去」「產品一定要 app」。
2. **列出這個結論背後的假設**：為什麼說不可能？基於什麼前提？哪些是事實，哪些是慣例？
3. **把每個假設拆到「不可再拆」**：
   - 物理限制（能量、質量、光速、材料密度）
   - 數學關係（預算、流量、轉換率的算術）
   - 人的基本需求（省時間、省力、連結、地位）
   - 規範 / 法律（哪些是硬限制，哪些只是慣例）
4. **用這些基本事實重新推演**：如果只有這些限制，結論還會一樣嗎？會不會出現不同的可能路徑？
5. **對照現狀找出「慣性」的部分**：現狀和推演出的結論差距在哪？那個差距往往就是機會。

## 案例

- **SpaceX 火箭成本**：業界說「火箭就是貴」。Musk 拆回「鋁合金、鈦、銅、碳纖維的原物料成本」——發現實際材料只佔傳統火箭售價的 2%。98% 是製程與供應鏈的慣性。於是重建垂直整合供應鏈，把成本壓到十分之一。
- **AWS 的誕生**：Bezos 問「企業為什麼一定要自己買伺服器？」拆回基本事實——他們需要的只是「計算資源」與「儲存」，不是「實體伺服器」。結論是把計算變成可租用的服務。
- **Netflix 訂閱制**：業界類比是「影片出租要照次計費」。Netflix 拆回「使用者要的是方便看片，不是計費方式」——於是做成吃到飽，後來又拆一次變成串流。每次都打破一層產業慣例。
- **日常應用**：猶豫要不要離職——類比思考是「同事都還在撐」「換工作風險高」。第一性原理：我每天工時 X 小時、薪水 Y、成長速度 Z、機會成本是 W，這組數字能不能換？

## 常見誤用

- **什麼事都要從頭推演**：癱瘓。第一性原理是關鍵決策工具，不是日常思維。多數日常事務用類比就夠快夠好。
- **誤把「不同於主流」當作第一性原理**：為反對而反對，沒有真的拆回事實。結果只是換了一個主觀偏好。
- **忽略類比推理的效率**：Bezos 說「大多數日常決策應該是類比 + 經驗」，只有少數決策值得第一性原理的成本。
- **拆到一半就停**：拆了兩層覺得「夠深了」就開始重建。通常真正的慣性假設藏在第三、第四層。
- **只質疑別人的假設，不質疑自己的**：自己的立場也建立在一組假設上，要一起拆。

## 與其他模型的關係

- **+ [MECE](mece.md)**：拆解時用 MECE 確保覆蓋完整、沒有重疊。
- **+ [5 Whys](five-whys.md)**：5 Whys 是往下問「為什麼」；第一性原理更像問「這個前提真的成立嗎」——前者追因果，後者審假設。
- **+ [decision 模組](../decision/index.md)的[二階思考](../decision/second-order-thinking.md)**：拆回事實後重新推演，二階思考幫你想「如果這樣改了，下一步世界會怎麼反應」。
- **+ [bias 模組](../bias/index.md)**：第一性原理最常被**[確認偏誤](../bias/confirmation-bias.md)、[從眾](../bias/bandwagon-effect.md)、權威偏誤**劫持。拆假設時要警覺自己是不是在合理化既有結論。
- **vs [類比推理](../learning/analogy.md)**：不是對立，是互補。快速決策靠類比，關鍵決策靠第一性原理。兩者錯配會出大問題。
