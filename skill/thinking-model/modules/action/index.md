# 四、行動與迭代

> 載入時機見 `SKILL.md` 的路由表。本檔為此類別的**子路由**——先對照下表判斷使用者的情境，再 Read 對應的 submodule 深入取用。

把想法變成結果、並在不確定中持續修正。七個 submodule 對應七種不同的行動情境，不要一次載入全部。

---

## Submodule 子路由表

### 1. MVP（最小可行產品） → `action/mvp.md`

**何時載入**：
- 使用者要做一個**新東西**（產品、功能、副業、活動），但還沒有人驗證過
- 花大量時間資源前，最關鍵的假設（有人要嗎？會付錢嗎？通路通嗎？）還沒測過
- 完美主義傾向導致遲遲不發布

聚焦：**驗證假設 > 完成作品**。

---

### 2. OODA 循環（Observe–Orient–Decide–Act） → `action/ooda.md`

**何時載入**：
- **競爭**情境（產品對打、談判、Q&A、比賽）或**快速變化**的環境（危機、市場劇變）
- 使用者陷入**分析癱瘓**，資訊永遠不夠、不敢行動
- 感覺被對手 / 局勢牽著走、「總慢半拍」

聚焦：**循環速度 > 單次完美**。強調 Orient（定位）這個多數人跳過的步驟。

---

### 3. PDCA 循環（Plan–Do–Check–Act） → `action/pdca.md`

**何時載入**：
- **流程 / 品質優化**：已有穩定活動在跑，要系統化改進（良率、速度、bug 率）
- **習慣、技能養成**需要可測量的迭代
- 使用者「一直在做但沒進步」——多半缺了 Check

聚焦：**每次循環累積知識**。跟 OODA 的差別：PDCA 對應穩定流程與長期改善，OODA 對應對抗與速度。

---

### 4. 複利思考（Compounding） → `action/compounding.md`

**何時載入**：
- **長期目標** 看不到進展，使用者開始懷疑或想放棄
- 使用者在追求短期爆發（「一個月賺 100 萬」「三週學會 X」）
- 評估**投資 / 學習 / 職涯**時要在「短期高報酬」和「長期累積」之間取捨
- 健康、關係、聲譽、信任等**長期變數**經營

聚焦：**時間是最被低估的變數**；避免歸零事件比追求最高報酬重要。

---

### 5. 槓桿思考（Leverage） → `action/leverage.md`

**何時載入**：
- 使用者覺得「每天很忙但影響力不大」、卡在**時間換錢**的線性模式
- 想提升**收入天花板**、規模化、跳脫 1:1 的工作結構
- 職涯 / 商業模式選擇：接案 vs SaaS vs 內容 vs 管理團隊

聚焦：**Naval 的四種槓桿**（勞力、資本、程式碼、媒體），後兩種是無許可槓桿（permissionless）。前提是 craft 已到位，否則槓桿會放大錯誤。

---

### 6. 時間盒（Timeboxing） → `action/timeboxing.md`

**何時載入**：
- **完美主義拖延**（「我想做到最好再發」）
- **開放式任務**（寫作、研究、設計、學習）容易無限延長
- **會議 / 討論**開不完、沒結論
- 使用者描述「這件事花了超過預期好幾倍的時間」

聚焦：**從任務驅動切換到時間驅動**。Parkinson's Law、強制取捨、保護深度工作。

---

### 7. 兩分鐘法則（Two-Minute Rule） → `action/two-minute-rule.md`

**何時載入**：
- **版本 A（GTD）**：使用者**待辦爆炸**、小事堆積、訊息回不完
- **版本 B（Atomic Habits）**：使用者**建立新習慣總是放棄**、all-or-nothing、啟動門檻高

聚焦：**降低啟動阻力**。版本 A 管既有待辦，版本 B 建新習慣，共同內核是「啟動是最大的阻力」。

---

## 挑選提示

- **一次性新事物** → MVP
- **不斷重複的流程** → PDCA
- **對抗或高速變化** → OODA
- **長期方向已定** → Compounding
- **要放大影響力** → Leverage
- **時間失控** → Timeboxing
- **開始太難 / 習慣建立** → Two-Minute Rule

多數使用者需求只對應一個 submodule。真的需要跨 submodule 時，典型組合：

- **新產品發布**：MVP（驗證）→ PDCA（迭代）→ Leverage（放大）
- **個人成長**：Two-Minute Rule（啟動）→ Compounding（累積）→ PDCA（精進）
- **工作效率**：Timeboxing（專注）+ Two-Minute Rule（清雜務）+ 80/20（prioritization 模組）
