# PDCA 循環（Plan–Do–Check–Act）

## 核心概念

PDCA 由 Walter Shewhart 提出、W. Edwards Deming 發揚光大，是品質管理與精實生產（Toyota Production System）的底層引擎。核心假設：**任何流程都有可改善的餘地**，只要把它變成一個可以測量、可以回饋、可以修正的循環。

四步驟：
1. **Plan（計畫）**：針對一個具體問題提出假設——「如果我改 X，Y 應該會變好」。寫下預期指標。
2. **Do（執行）**：在**小範圍**試行（不是全面 rollout），收集資料。
3. **Check（檢核）**：比對實際結果和當初的預期。差距在哪？為什麼？
4. **Act（調整）**：
   - 有效 → 標準化、推廣；
   - 無效 → 修正假設，回到 Plan。

關鍵不是「跑一次」，而是**每次循環都累積下一次的起點**。Toyota 幾十年的競爭力來自每個作業員每天都在跑自己崗位上的 PDCA。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- **流程優化**：現有流程在跑，但有地方卡卡的、良率不夠、速度太慢
- **品質管理**：產出的結果（bug 率、客訴、交期）需要系統化改善
- **習慣養成**：想建立新習慣、想戒掉壞習慣，需要可測量的迭代
- **學習語言 / 技能**：重複練習 + 回饋 + 調整，正是 PDCA 的核心
- 使用者描述「我一直在做，但沒看到進步」——往往是少了 Check 這步

不適合時機：
- 在**對抗 / 快速變化**的情境 → 用 [OODA](ooda.md)
- 要驗證未知假設 → 用 [MVP](mvp.md)
- 一次性、非重複的任務 → PDCA 沒有發揮空間

## 如何應用

1. **選一個具體、可測量的問題**：「想變強」不是 PDCA 問題，「每週部署 bug 率想從 15% 降到 5%」才是。
2. **Plan 時寫下數字**：預期指標 + 達成時限。沒有數字的 Plan 後面會 Check 不動。
3. **Do 時縮小範圍**：先在一個 team / 一個產品線 / 一個月試，不要全公司一起改。
4. **Check 要誠實**：最難的一步。多數失敗的 PDCA 不是敗在方法錯，而是 Check 時自我欺騙（「其實差不多吧」「這次特殊情況」）。
5. **Act 要留下紀錄**：無論成功失敗，把學到的東西寫下來。下次循環才有複利效應。
6. **循環要夠短**：半年才檢一次的 PDCA，等到修正時問題已經擴散。週 / 月尺度通常比較剛好。

## 案例

- **Toyota 的安燈繩**：作業員發現問題立刻拉繩停線，每一次停線都是一次 PDCA 起點。短循環、高頻率、累積成世界級品質。
- **程式團隊的 retrospective**：每 sprint 結束檢討（Check）→ 決定下 sprint 要調整什麼（Act → Plan）→ 下 sprint 執行（Do）→ 下次 retro 再 Check。
- **減肥**：Plan「每週運動 3 次、減醣」→ Do 兩週 → Check 體重 / 體脂變化 → Act（有效就持續、無效就調整策略）。失敗案例多半是 Check 沒量體重、全憑感覺。
- **部落格寫作**：Plan「這篇嘗試用案例開頭」→ Do → Check 閱讀數、完讀率 → Act 調整下一篇的開頭策略。

## 常見誤用

- **跳過 Check**：只有 Plan + Do，永遠不回頭看。一年後「好像有進步」但說不上來哪裡，也無法複製。
- **Check 但不 Act**：檢討完報告寫得漂漂亮亮，然後下週照原樣過。累積無數份「事後檢討報告」毫無行為改變。
- **Plan 太大**：一次想改 10 件事，Check 時無法判斷哪個變因造成影響。一次一個變因，才能學到東西。
- **把 PDCA 當官樣文章**：當成要交差的流程，就失去它最有價值的部分——真誠地檢視、真誠地修正。

## 與其他模型的關係

- **vs [OODA](ooda.md)**：PDCA 對應穩定流程、長期改善；OODA 對應對抗、高速變動。
- **+ [複利思考](compounding.md)**：PDCA 每次循環的 1% 改善，長期複利成巨大差距。
- **+ [刻意練習](../learning/deliberate-practice.md)**（[learning 模組](../learning/index.md)）：刻意練習本質上是 PDCA 在技能上的特化——鎖定弱點（Plan）、練習（Do）、教練或錄影回饋（Check）、調整（Act）。
