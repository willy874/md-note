# OODA 循環（Observe–Orient–Decide–Act）

## 核心概念

OODA 由美國空軍上校 John Boyd 在研究空戰時提出。他發現：**贏的不一定是飛機性能更強的一方，而是決策循環更快的一方**。當你的循環比對手快，你看到的世界已經更新了，對手的決策建立在過期情報上，等於在跟「你上一個版本的你」作戰。

四步驟：
1. **Observe（觀察）**：收集外在情境（資料、對手動作、環境變化）。
2. **Orient（定位）**：把觀察放進**自己的背景、文化、經驗、偏誤**中解讀。Boyd 認為這步最關鍵，也最容易被忽略——同樣的情報，不同人會得出完全不同的「這是什麼局面」。
3. **Decide（決策）**：基於當下的定位選行動。
4. **Act（行動）**：執行，然後立刻回到 Observe，看世界如何回應。

關鍵不是「循環一次多快」，而是**比對手快**。相對速度決定優勢。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- **競爭情境**：產品對打、談判、簡報 Q&A、運動比賽、爭取資源
- **快速變化**：市場劇變、危機處理、突發狀況、黑天鵝
- **資訊不完整但必須行動**：等不到「收齊資料」那天；再等就輸了
- 使用者陷入**分析癱瘓**（analysis paralysis），一直在規劃從不行動
- 使用者被對手 / 市場牽著走，總覺得「慢半拍」

不適合時機：
- 沒有對手、沒有急迫性的長期任務 → 用 [PDCA](pdca.md)
- 需要驗證未知假設 → 用 [MVP](mvp.md)
- 個人習慣養成 → 用 [compounding](compounding.md) 或 [two-minute rule](two-minute-rule.md)

## 如何應用

1. **縮短 Observe 的延遲**：問自己「我看到的是幾天前的情報？」。建立能即時反映現狀的儀表板 / 指標。
2. **意識到 Orient 的偏誤**：同樣一張圖表，經驗不同的人讀出的故事完全不同。刻意尋找**反面解讀**（參考 [bias 模組](../bias/index.md)的[確認偏誤](../bias/confirmation-bias.md)、[Steelman](../learning/steelman.md)）。
3. **容忍「夠好就好」的決策**：等到「完美決策」時，世界已經換檔三次了。70 分現在做 > 95 分太晚做。
4. **把 Act 設計得便宜**：行動後能快速收到回饋，而不是砸大筆資源看半年後才知道對錯。這一點和 [MVP](mvp.md) 相通。
5. **刻意加速對手的 Orient**：在競爭情境中，讓對手的情報過期、讓他的解讀混亂，就能瓦解他的循環。這就是 Boyd 所謂的「getting inside the opponent's OODA loop」。

## 案例

- **空戰原型**：F-86 速度其實不如 MiG-15，但座艙視野好（Observe 快）、操作直覺（Decide–Act 快），結果對戰勝率壓倒性。
- **產品功能戰**：小團隊對大公司的最大武器是循環速度——每週發版 vs 每季發版，一年下來你迭代 52 次對方 4 次。
- **會議 Q&A**：對方拋出刁鑽問題時，強者不是答得最完美，而是**快速定位「他真正在問什麼」**，然後給出夠用的答案，繼續推進。
- **日常工作**：收到合作方丟來的訊息 → 觀察（他真的在抱怨什麼？）→ 定位（背後是 KPI 壓力還是情緒？）→ 決策 → 回覆。多數人跳過 Orient 直接反射性回應，引爆衝突。

## 常見誤用

- **跳過 Orient**：從觀察直接到決策，等於用反射神經做事，容易被情緒與既有偏誤劫持。Orient 是 OODA 的靈魂。
- **把 OODA 當 PDCA 用**：OODA 強調的是**在對抗 / 不確定中搶速度**，[PDCA](pdca.md) 強調的是**在穩定流程中累積品質**。情境錯了，工具就沒用。
- **只追求快**：快但方向錯的循環只是加速撞牆。Orient 一旦歪掉，循環越多次傷害越大。
- **沒有明確的對手或環境變數**：OODA 最吃力不討好的場景是「一個人獨自規劃長期目標」——這時 [PDCA](pdca.md)、[compounding](compounding.md) 更合適。

## 與其他模型的關係

- **vs [PDCA](pdca.md)**：OODA 對應不確定 / 對抗環境，PDCA 對應已知流程的持續優化。現實中常常交替使用。
- **+ [bias 模組](../bias/index.md)**：Orient 非常容易被[確認偏誤](../bias/confirmation-bias.md)、[錨定效應](../bias/anchoring.md)劫持，做重要決策時要交叉檢查。
- **+ [decision 模組](../decision/index.md)**：當一步是不可逆決策時，OODA 的 Decide 要慢下來（配合 [Pre-mortem](../decision/pre-mortem.md)），其他時候可以快。
