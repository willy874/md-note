# 類比思考（Analogical Thinking）

## 核心概念

**用熟悉領域的結構，去理解陌生領域。** 大腦記憶新資訊最有效的方式，是把它「掛」到已知的結構上——這就是類比的力量。

好的類比有三個層次：
1. **表層相似**（膚淺）：形狀、顏色、名字像——沒什麼用。
2. **結構相似**（有用）：因果關係、運作機制、角色配置一致——這才是真正的類比。
3. **深層原理**（最強）：兩件事其實遵循同一條底層規律——這種類比會讓人大笑「原來是同一件事！」

經典例子：Rutherford 的原子模型「像一個小太陽系」——電子繞核如行星繞恆星。雖然量子力學後來推翻了這個圖像，但結構類比讓好幾代學生「進得去」原子物理。

類比的危險：**一旦兩者的結構在某個面向分岔，盲目套用會誤導。** 所以類比是**入門工具**，不是**推論工具**——它幫你理解，不負責替你做精確判斷。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者**跨入新領域**，現有知識和新領域看似無關，不知從何下手
- 要**教非專業人**理解技術 / 複雜概念（或把非技術人的需求翻譯給工程師）
- **創意發想**、命名、產品定位、slogan——需要借用一個熟悉結構
- 對一個新概念「卡住進不去」，純讀教科書式定義效果不好

不適合時機：
- 需要**精確推論 / 嚴謹證明** → 類比會失真
- 概念已經足夠熟悉 → 類比反而多此一舉

## 如何應用

1. **先找「它在做什麼」**：不是問「它像什麼」，而是先問「它的功能 / 角色 / 因果關係是什麼」。
2. **在熟悉領域找同構物**：有沒有一個你熟的東西，角色配置相同？例：
   - DNS → 網路電話簿
   - Git branch → 寫小說時的「如果⋯⋯分支」
   - React 的 state → 白板上的一塊區域，有人改了就要重畫
3. **講出來、測試對方能不能接續推論**：好類比讓人能「用熟悉結構往前推」——對方能自己問出下一個問題，就對了。
4. **明確標出「類比失效的地方」**：用到一半，告訴對方「但這裡就不一樣了——原子核不像太陽，因為⋯⋯」。這個「誠實切斷」的動作讓對方對新領域的認知更準。
5. **累積類比庫**：好講者 / 好老師腦中有一大堆「跨領域類比對」，遇到解釋場景能快速調用。

## 案例

- **解釋快取（cache）**：「像你把常看的書放在書桌上，不常看的放書櫃——書桌拿比較快，但空間小」。對方立刻懂「快取 vs 記憶體」的取捨。
- **教 API**：「API 像餐廳菜單 + 服務生——你不需要知道廚房怎麼煮，只要照菜單點，服務生會把東西端上來」。
- **跨領域創意**：Steve Jobs 把 iPod 定位成「1000 首歌放口袋」——類比自一個熟悉的場景（隨身聽），而非技術規格。
- **教 LLM**：「像一個讀了整個圖書館的朋友，被問到什麼都會接話——但他不『知道』事實，只是根據讀過的東西猜最合理的下一句」。

## 常見誤用

- **依賴錯誤類比做決策**：用「大腦像電腦」去推論人類記憶機制——在結構上有很多分岔,推論會錯。類比只入門,不推論。
- **用表層相似而非結構相似**：「加密貨幣就像鬱金香泡沫啊」——表面上都是投機,但經濟結構其實差很多。盲用這種類比會下錯判斷。
- **類比無限延伸**：一個類比用到 80% 很好,硬撐到 120% 就壞了——類比有適用範圍,超出就要換。
- **自己沒懂就用類比逃避**：有時人用類比是因為自己也沒真懂。類比應該是**懂了之後的教學工具**,不是**掩飾不懂的遮羞布**。

## 與其他模型的關係

- **+ 費曼技巧**：費曼最常用類比做白話化。兩者相輔相成——費曼是方法論,類比是工具。
- **+ 最小可學單位**：新概念進來時,找一個舊 atomic concept 做類比,連結速度最快。
- **+ 受眾思維**：類比要好,必須先知道受眾熟悉什麼——同一個概念對工程師、投資人、小孩要用不同類比。
- **+ 第一性原理**（analysis 模組）：類比和第一性原理是學習的兩端——類比是「借用熟悉結構」,第一性原理是「拆到最底層重新組裝」。複雜問題常常兩者交替用。
