# 刻意練習（Deliberate Practice）

## 核心概念

由瑞典心理學家 Anders Ericsson 研究頂尖表現者（小提琴家、西洋棋手、運動員）提出。他發現**「天才」神話其實建立在大量刻意練習之上**——不是 10,000 小時的魔法，而是這一萬小時的**性質**：目標清楚、在能力邊緣、即時回饋、反覆修正。

三個能力區：
1. **熟練區**：做已經會的事，舒適但幾乎零進步——多數人的「練習」其實在這裡。
2. **學習區**：剛超出現有能力一點，做的時候會卡、會累、會不舒服——**真正的成長都在這區**。
3. **恐慌區**：難度太高無法消化，練了也無效，還會挫敗退坑。

Ericsson 的關鍵發現：**頂尖選手 vs 業餘選手的總練習時間其實差不多，關鍵差在「學習區」的時數**。1 小時刻意 > 8 小時散漫。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者**練很久但卡在同一個水平**（高原期 / plateau）
- 想把某項技能從「會」升到「厲害」，不只是業餘玩玩
- 樂器、程式、寫作、運動、外語口說、設計、銷售等**技能型**追求
- 每次練都很舒服、沒出錯——這是沒進步的警訊

不適合時機：
- 概念理解為主的學習 → 用費曼 / 最小可學單位
- 啟動都還沒啟動 → 先用兩分鐘法則建立 streak

## 如何應用

1. **定義「好」的樣子**：你追的卓越長什麼樣？有具體可觀察的指標 / 範本嗎？
2. **拆技能樹**：總能力太大，拆成可練的子技能（寫作 → 開場鉤子、段落銜接、收尾；寫程式 → 除錯、命名、架構設計）。
3. **找到弱點**：哪個子技能拖垮整體？優先練它（與 80/20 一致）。
4. **在能力邊緣設計題目**：比現在水平**稍微**難一點，做完有點累但能完成。太簡單 = 熟練區，太難 = 恐慌區。
5. **即時回饋**：老師、教練、資深同事、錄影回放、評分系統、比賽結果——沒有回饋的練習等於盲練。
6. **小循環迭代**：練 → 得到回饋 → 調整 → 再練。配合 PDCA 循環。
7. **主動追求不舒服**：舒服的練習幾乎等於沒練。

## 案例

- **小提琴家**：頂尖學生每週單獨練習 20+ 小時，針對老師指出的弱點反覆練同一段 20 分鐘——而非「把整首曲子從頭拉到尾」。
- **程式設計**：刻意挑「還不會」的題目（LeetCode Hard、新語言、新 paradigm），而不是每天重寫已經熟的 CRUD。
- **寫作**：挑一個你寫不好的文類（技術說明、情緒敘事、說服文），連續寫 20 篇，每篇後找人回饋、標出弱點。
- **銷售**：錄下自己的通話，重聽找「客戶猶豫的那一秒我說了什麼」，隔週演練 3 種更好的回應。

## 常見誤用

- **把「時數」當成刻意練習**：累積 10,000 小時但都在熟練區原地踏步,不會變頂尖。時數不是魔法,性質才是。
- **沒有回饋就開練**：自己想自己改,容易走火入魔。教練 / 同儕 / 客觀指標缺一不可。
- **一次練太多子技能**：每次練習專注 1–2 個弱點,不要同時調十件事。
- **逃避不舒服**：練習時感到沮喪、卡住、焦慮是正常的——沒這些感覺反而該檢查自己是不是混在熟練區。
- **誤以為適用所有領域**：對「有明確規則、可回饋」的領域（棋、樂器、運動、程式）最有效;對「目標模糊」的創意工作,效果遞減。

## 與其他模型的關係

- **+ PDCA**：刻意練習的每次小循環本身就是 PDCA（Plan 練什麼、Do 練、Check 結果、Act 調整）。
- **+ 複利**：刻意練習是複利最佳的載體——每次 1% 的修正,一年下來變另一個人。
- **+ 費曼**：練「理解」用費曼,練「技能」用刻意練習。兩種學習的主武器,分工清楚。
- **+ 80/20**（prioritization 模組）：花 80% 的練習時間在拖累整體的 20% 弱點上。
