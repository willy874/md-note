# 可得性偏誤（Availability Heuristic）

## 核心概念

可得性偏誤由 Daniel Kahneman 和 Amos Tversky 在 1970 年代提出：**我們估計一件事的「機率 / 頻率」時，實際上在衡量的是「這件事的例子有多容易想到」**。

容易想到的例子會被高估，難想到的會被低估。影響記憶可得性的因素：

- **近期性**：剛發生 / 剛看到 → 想得到
- **戲劇性**：恐怖、鮮明、情緒強 → 想得到
- **個人相關**：發生在自己或親友身上 → 想得到
- **重複曝光**：新聞、演算法推送反覆看到 → 想得到

結果是我們對**戲劇性、罕見、近期**的風險過度擔心，對**平淡、高頻、慢性**的風險嚴重低估。

核心洞見：**腦袋裡調出來的樣本 ≠ 世界真實的分布**。演化上這是合理的捷思（最近的威脅 = 最需要應付的），但在現代資訊環境（新聞 + 演算法）下會嚴重失真。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者用「最近好多人⋯⋯」「前幾天才看到⋯⋯」「朋友的朋友⋯⋯」當**通則**
- 對**戲劇性風險**（空難、綁架、食安）過度擔心；對**無聊高風險**（車禍、久坐、高血壓）漠然
- 剛發生某事後，覺得「到處都在發生」（懷孕後發現街上都是孕婦、買了某車滿街都看到）
- 根據**新聞熱度**調整行為或投資——媒體在意什麼 ≠ 統計上重要的什麼
- 找工作 / 選科系時參考**身邊少數例子**而不是整體就業資料

不適合時機：

- 使用者已經在**引用統計數據**——可能是[確認偏誤](confirmation-bias.md)等其他偏誤，不是可得性
- 討論的是**個人 / 小樣本**（「我這家店今天賣了幾杯」）——小樣本本來就該用個人經驗

## 如何應用

1. **發現「腦海樣本」和「真實資料」的差距**：直覺給一個數字 → 查實際統計 → 比對差多少。多數人會被差距震驚。
2. **問自己：「我這個印象是靠幾個例子建立的？」** 三個？五個？對比母群體是幾千幾萬。
3. **刻意接觸「無聲多數」**：新聞只報異常，所以只讀新聞的人會嚴重偏誤。去讀**統計年鑑 / 基準率（base rate）**，不是只讀故事。
4. **區分「記得容易」和「真的常見」**：
   - 鯊魚 vs 自動販賣機：哪個殺的人多？（答案：自販機，但新聞只報鯊魚）
   - 空難 vs 車禍：哪個死亡率高？（車禍是空難的數十倍，但新聞反過來）
5. **對「熱點」事件後的決策降速**：重大新聞後的一週內，判斷力會被劫持。不要在此時做大決定（投資、買房、移民）。
6. **建立個人「基準率檔案」**：常用議題（就業、疾病、犯罪）的真實基準率是多少？記下來，遇到新聞衝擊時對照。

## 案例

- **恐攻 vs 車禍**：911 後一年，美國人因為怕飛而改開車，估計多出約 1,600 人死於車禍——比恐攻本身死傷還高。怕飛不怕開車，就是可得性偏誤。
- **疫苗副作用**：社群上看到一則嚴重副作用的貼文（戲劇 + 情緒），對「幾百萬分之一」的機率產生「很常見」的錯覺。拒打造成的風險反而被低估。
- **創業成功率**：只讀 Elon Musk、Patrick Collison 的故事，覺得「創業就是有機會」。實際上 90% 新創五年內倒閉——但失敗者沒故事，所以你想不到。（這也和[倖存者偏誤](survivorship-bias.md)疊加。）
- **投資**：某股上新聞漲停三次，你覺得「它常常漲」，忽略之前 300 天的平淡走勢。新聞偏誤 → 可得性 → 進場追高。
- **找工作**：有個學弟畢業一年進 Google，你覺得「這間還容易進」。忽略每年數萬應徵、錄取率個位數的基準率。
- **治安感知**：新聞連報三起社會案件，你覺得「最近治安變差」。其實犯罪率統計可能持平甚至下降——只是媒體這週編排剛好集中。

## 常見誤用

- **以為「反直覺」一定對**：發現自己有可得性偏誤後，反過來否定所有直覺。但有時你的直覺是對的（小樣本確實在變化）。工具是校準，不是反轉。
- **用統計數字否定個人經驗**：統計適用於母群體，但個人決策有時就是看你那個小樣本。平均值不代表你的情境。要區分「通則判斷」vs「個案決策」。
- **過度追新聞去修正**：發現自己受媒體影響後，改追更小眾的消息——仍然是可得性，只是換了一組樣本。真正的對策是看**統計**，不是換個故事來源。
- **用「這只是可得性」打發真實風險**：有時新聞反覆報一件事，是因為它真的值得擔心（例：新興傳染病早期）。要看資料，不要用偏誤標籤草率否定。
- **忽略「近期性」也可能有訊號**：最近發生的趨勢有時確實代表結構變化。不能一概用「可得性」撇開。需要的是「是趨勢還是雜訊」的檢驗。

## 與其他模型的關係

- **+ [確認偏誤](confirmation-bias.md)**：容易想到的例子剛好支持你原本相信的 → 兩個偏誤互相強化成封閉迴路。
- **+ [倖存者偏誤](survivorship-bias.md)**：能被「想到」的多半是成功、戲劇性、被報導的——失敗者既沒被記錄也沒人會講。
- **+ [敘事謬誤](narrative-fallacy.md)**：可得的例子往往是好故事。可得性供應材料，敘事謬誤把它編成因果。
- **+ [從眾效應](bandwagon-effect.md)**：大家在談什麼 → 那件事變得可得 → 你覺得它更重要 → 你也開始談。
- **vs 統計思維 / 基準率思維**：是最直接的解毒劑。凡是估機率前先查 base rate。
- **+ [decision 模組](../decision/index.md)的[期望值](../decision/expected-value.md)**：期望值計算強迫你用真實機率而非腦中印象。
- **+ [系統思考](../analysis/systems-thinking.md)**（[analysis 模組](../analysis/index.md)）：媒體 → 情緒 → 行為 → 更多新聞，是一個可得性驅動的增強迴路。
