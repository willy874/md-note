# 確認偏誤（Confirmation Bias）

## 核心概念

確認偏誤是指人**搜尋、解讀、記憶資訊時，不自覺地偏向支持自己既有信念的那一邊**。由心理學家 Peter Wason 在 1960 年代的實驗中首次系統化（著名的 2-4-6 任務）——受試者被要求找出一個數列規則，幾乎所有人只嘗試「符合自己猜測」的例子，極少人嘗試能**推翻**自己的例子。

三個作用層：

1. **搜尋偏誤**：選擇性地 google、選擇性地問朋友、選擇性地讀新聞。
2. **解讀偏誤**：同一份資料，不同立場的人讀出不同結論。模糊證據被塞進既有框架。
3. **記憶偏誤**：支持自己觀點的經歷記得牢，反例幾週後就忘了。

核心洞見：**一個反例的資訊量 > 十個支持證據**。科學進步不是靠累積支持案例，而是靠**可證偽的反例**（Popper）。日常思考多數人做反了——累積一堆「我就說吧」，從不主動找推翻自己的機會。

確認偏誤的威力在於**它不會被意識到**。你會覺得自己「只是在看事實」，但其實事實已經被你過濾了。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者只引用**支持自己結論**的資料，對反例「視而不見」或快速否定
- 研究 / 投資 / 醫療診斷 / debug 時，**只追一個假設**，其他可能性從未被認真測試
- 討論議題時，使用者蒐集的證據**全部偏向同一邊**（社群演算法推波助瀾）
- 決策完後不斷尋找「這個決定是對的」的證據（已成定局仍在自我說服）
- 爭論中從不出現「我以為是 X，查了資料發現是 Y」這種修正

不適合時機：

- 使用者**已經在主動找反例**、做嚴謹對照——那是在對抗偏誤而非被偏誤抓住
- 議題**沒有足夠反方資料存在**（新興領域、稀有事件）——不是偏誤造成的，是資料本身偏

## 如何應用

1. **寫下「什麼證據會讓我改變看法」**（pre-registration）：下結論前就寫。事後再補會被合理化。
2. **主動尋找反例**：花至少 30% 的研究時間找「我可能錯在哪」。具體做法：
   - Google 搜「X is wrong」「problems with X」
   - 讀持相反立場的權威文章（不是稻草人版本）
   - 找跟你立場不同的人聊 30 分鐘，認真聽
3. **做 Steelman**：把對方論點重建成**最強版本**再反駁。如果你反駁不贏最強版，你就還沒準備好下結論。
4. **Red Team / Pre-mortem**：刻意找人（或自己扮演）挑戰你的方案。Pre-mortem 是「假設這個決定三個月後失敗了，為什麼？」
5. **改變搜尋語言**：不要查「為什麼 X 是對的」，要查「X 的限制是什麼」「X 什麼時候不適用」。
6. **事後回看**：記錄過去的判斷與結果。哪些次你其實忽略了當時就存在的反證？模式會浮現。

## 案例

- **Wason 2-4-6 任務**：規則是「任何遞增數列」，但受試者多數猜「每次 +2」，然後只丟 6-8-10、10-12-14 這種符合自己猜測的——全部被判對，還是沒找出真規則。只有丟 2-4-5、3-1-5 這種**反例測試**的人才找得出來。
- **投資**：買進某股後，只讀這檔的多頭分析，推特只追唱多帳號。股價逆勢下跌被解讀為「市場錯」，繼續加碼——經典確認偏誤放大損失。對策：強迫自己每天讀一篇唱空。
- **醫療診斷**：第一個假設是流感，後續所有症狀都被塞進流感解釋，結果延誤診斷真正的結核 / 心肌炎。醫學教育有一套 "differential diagnosis"（鑑別診斷）流程就是對抗這個偏誤。
- **debug**：「一定是那個新 commit 的問題」——花三小時在新 commit 上找 bug，其實是前一天的 config 改動。對策：先做二分搜尋 / 回滾驗證，不要只追自己懷疑的那條線。
- **政治 / 議題**：FB / YouTube 演算法把你同溫層推到極致，支持自己立場的內容來得又快又多。一年後你會覺得「反方都瘋了」，其實是你沒讀到他們的版本。
- **感情**：分手時只記得對方讓你痛苦的事，忘記開心的時刻；復合時反過來。記憶本身已被偏誤過濾。

## 常見誤用

- **以為自己沒有**：確認偏誤最狡猾之處就是它不會讓你意識到。覺得「我很客觀」的人往往最受影響。對策是流程（預先寫下判準），不是自我感覺。
- **把 Steelman 做成稻草人**：嘴上說「我來考慮反方」，實際上把反方論點簡化到弱版再反駁。沒有真的挑戰自己。
- **只在已輸的戰場搜尋反例**：方案已經訂了才補做 pre-mortem，只是形式主義。要在**決策前**做。
- **全盤反轉成懷疑一切**：從「只看支持」跳到「什麼都不信」，另一種失能。目標是**校準**，不是癱瘓。
- **用確認偏誤當擋箭牌**：別人提出反例時回一句「你有確認偏誤」就草草結案，反而是自己最不想面對反證。

## 與其他模型的關係

- **+ 倖存者偏誤**：兩者常疊加——只看成功者（倖存者）+ 只看支持自己方法的成功者（確認）。
- **+ 可得性偏誤**：你最近看到的就是你願意相信的證據。兩者互相強化。
- **+ 從眾效應**：大家都這樣想 → 你找到支持共識的證據就停下來，沒去找反例。
- **+ 敘事謬誤**：有故事框架後，所有新資料都被塞進故事。故事是確認偏誤的容器。
- **vs 第一性原理**（analysis 模組）：第一性原理強迫你拆回事實重建，是對確認偏誤最強的解毒劑之一。
- **+ decision 模組的 Pre-mortem**：Pre-mortem 是系統化找反例的決策工具。
