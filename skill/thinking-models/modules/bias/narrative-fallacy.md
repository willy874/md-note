# 敘事謬誤（Narrative Fallacy）

## 核心概念

敘事謬誤由 Nassim Taleb 在《黑天鵝》中系統化：**人類大腦會把一連串隨機 / 複雜的事件，壓縮成一個有因有果的故事，因為故事比事實容易記、容易講、容易信**。

大腦是**故事機器**，不是**統計機器**。給我們一堆點，我們會自動連成線、賦予主角、加入動機、找出轉折。這在演化上有用——故事方便記憶和傳承——但在分析隨機性、評估決策、看待成敗時會誤導。

敘事謬誤的三個典型表現：

1. **事後諸葛（hindsight bias）**：用現在知道的結果，倒推出「其實當時就看得出來」。市場崩盤後每個分析師都說「我早就警告了」。
2. **英雄式傳記**：成功者的人生被編成「天賦 + 關鍵選擇 + 奮鬥 → 必然成功」的故事，把運氣成分抹掉。
3. **為雜訊賦義**：不相關的事件被串成因果（「那次會議他沒笑，一定是在氣我」）。

Taleb 的測試：**用**更少**的字講同一段歷史，聽起來一樣合理、一樣必然——這就是故事的魔術，它把資訊壓縮到失真**。

核心洞見：**不是每件事都有原因，也不是每個成功都有祕訣**。有些事就是**雜訊**。承認雜訊存在，是辨識真正訊號的前提。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者把**隨機事件**講成有因有果的故事（「當年若不是那通電話我就⋯⋯」）
- **事後諸葛**：用現在的結果倒推「我早該看出來」
- 評估**投資、產品、政策、人生選擇**時，被**動人的故事 / 傳記 / 案例**說服而忽略機率
- 想**複製**某個成功人士的路徑（「他三步驟達成 X，我也照做」）
- 對**複雜歷史 / 市場變化**給出單一原因（「這次崩盤就是因為 X」）
- 關係衝突中把對方的行為串成「他就是針對我」的劇本

不適合時機：

- 因果**真的清晰**（實驗室可驗證的機制）——那不是敘事謬誤，是科學
- 敘事作為**溝通 / 學習 / 記憶** 的工具時——故事在學習情境有正面價值，只要使用者知道「這是簡化」

## 如何應用

1. **區分「資訊」與「故事」**：任何看似合理的敘事，問：「把結論抽掉、只剩資料，我還能推出同樣結論嗎？」多數故事拿掉結論後就不成立。
2. **承認「雜訊」的存在**：並非每個波動都有原因。每日股價有 80% 是雜訊；每次對話的微表情不一定是訊號。不要每件事都解讀。
3. **Pre-mortem + Post-mortem 比較**：事前寫下你對事件的預測；事後回看與實際結果的差距。這能戳破「我早就知道」的事後諸葛幻覺。
4. **找反事實**：對每個「因為 X 所以 Y」，問「如果 X 換成 X'，Y 還會發生嗎？」多數時候會，因為真實因果有多個路徑。
5. **警覺「動人傳記」**：傳記、創業故事、TED 演講都是精心剪裁的敘事。其價值在**啟發**，不在**模板**。讀完問：「這些情節裡有多少是運氣？有多少是倖存者視角？」
6. **用機率語言取代故事語言**：「他應該會來」→「他來的機率大約 60%」。這個小動作會破除很多假故事。
7. **多版本敘事**：對同一件事寫三個不同的故事解釋。多數時候你會發現三個都說得通——表示原本那個故事不是唯一合理解。
8. **接受「有些事沒意義」**：不是每個挫折都是伏筆、不是每段經歷都有 takeaway。有時就是運氣差 / 運氣好。

## 案例

- **股市分析**：每天傍晚新聞「今天台股因 X 因素下跌 0.3%」。實際上多數小幅波動是雜訊，沒有單一可解釋原因。但每天都要有故事填版面，於是隨便挑一個當天新聞當解釋。
- **創業傳記**：「Jobs 因為在 Reed College 旁聽書法課，後來 Mac 的字體才好看」——聽起來很有伏筆。但他當年旁聽了很多課，多數沒用；字體好不好看和公司成不成功也未必是關鍵。故事把無數隨機事件剪成一條必然線。
- **事後諸葛（2008 金融危機）**：崩盤後，「我早就看出來」的分析師暴增。但**事前**真正有明確預測並持續押注成功的人極少（例：Michael Burry 等）。多數是事後重構故事。
- **產品成功歸因**：「Airbnb 靠在設計師身上下功夫才起來」、「Slack 靠早期的 onboarding 才贏」——這些都有道理，但**很多做了同樣事情的公司失敗了**（倖存者偏誤 + 敘事謬誤的雙重）。
- **關係衝突**：「他今天沒回我訊息 → 他最近都這樣 → 他在疏遠我 → 他其實從一開始就⋯⋯」。從一個資料點（沒回訊息）推出完整的劇本，忽略他可能只是忙。
- **歷史教科書**：「一戰是因為費迪南大公被刺殺」——真相是幾十年結構性緊張早已箭在弦上，刺殺只是最後一根稻草。但小說式敘事比結構分析好記。
- **人生轉折自述**：「當年若不是那通電話，我現在⋯⋯」。這類語句非常動人，但把幾千個影響因素簡化成一個——就是敘事謬誤。
- **棒球 / 運動比賽後的評論**：「他今天打不好是因為昨晚睡不好」——可能有關，但一場比賽的表現有大量隨機成分。用一個變數解釋等於忽略了分布。

## 常見誤用

- **變成什麼故事都不信的虛無**：發現敘事謬誤後，拒絕所有敘事。但人要行動，就需要某種敘事（價值觀、意義）。要**有意識地使用故事**，不是拒絕故事。
- **用「敘事謬誤」否定真實因果**：有些事真的有明確因果機制。不能遇到所有因果主張都回「這是敘事謬誤」。關鍵是**有沒有獨立證據**，不是「它看起來像個故事」。
- **自己用敘事、別人用就批判**：對自己的人生故事敘述很流暢，對別人的說「你這是敘事謬誤」。雙標。
- **把「隨機」當萬用擋箭牌**：「這次賠錢是隨機」——可能是，可能不是。要看證據、看模式、看分布，不是嘴上貼標籤。
- **教學 / 溝通場合也拒絕敘事**：故事是最強的記憶載體。在教學、演講、說服情境中，要用故事但同時標記「這是範例不是通則」。

## 與其他模型的關係

- **+ 倖存者偏誤**：倖存者提供的故事 + 敘事謬誤把故事包裝成公式。兩者合起來是「成功學書籍」的主要原料。
- **+ 確認偏誤**：一旦接受某個敘事，後續資料都被解讀進這個敘事框架。敘事 → 確認 → 更強的敘事。
- **+ 可得性偏誤**：生動的故事比統計容易記憶 → 敘事主導我們腦中的樣本。
- **+ 過度自信**：相信自己能從故事萃取「成功公式」，於是對複製成功過度自信。
- **+ 基本歸因謬誤**：把別人的人生壓縮成「他就是怎樣的人」也是一種敘事——用人格當主角，忽略情境支線。
- **vs decision 模組的期望值 / 機率思考**：敘事是點，期望值是分布。訓練自己用分布看事情，是敘事謬誤的主要解毒劑。
- **+ 第一性原理**（analysis 模組）：拆回事實重建，戳破「聽起來合理」的故事。
- **+ 系統思考**（analysis 模組）：系統圖強迫你看多變數互動，無法簡化成單一因果故事。
