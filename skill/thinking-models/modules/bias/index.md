# 六、避開認知陷阱

> 載入時機見 `SKILL.md` 的路由表。本檔為此類別的**子路由**——先對照下表判斷使用者的情境，再 Read 對應的 submodule 深入取用。

我們的大腦演化自訊息稀缺、風險立即的環境，內建一套快速但會出錯的判斷機制。**偏誤不是笨，是大腦的省電模式**——認識它們，才有機會在重要時刻多花一點電，繞過它們。

十個 submodule 對應十種常見偏誤，不要一次載入全部。

---

## Submodule 子路由表

### 1. 確認偏誤（Confirmation Bias） → `bias/confirmation-bias.md`

**何時載入**：

- 使用者只引用**支持自己結論**的證據，對反例視而不見
- 研究 / 投資 / 診斷時只往**一個假設**追，沒有測試替代假設
- 討論時只追問「為什麼我是對的」而不是「我可能錯在哪」

聚焦：**主動尋找反例**。一個反例的資訊量 > 十個支持證據。

---

### 2. 可得性偏誤（Availability Heuristic） → `bias/availability-heuristic.md`

**何時載入**：

- 使用者用**最近看到的新聞 / 朋友案例**當通則（「最近好多人⋯⋯」）
- 對**戲劇性風險**（空難、罕見病）過度擔心，對**無聊高風險**（車禍、久坐）漠然
- 剛經歷某事就覺得「到處都在發生」

聚焦：**看統計，不看故事**。生動 ≠ 高頻。

---

### 3. 錨定效應（Anchoring） → `bias/anchoring.md`

**何時載入**：

- **談判、估價、預算、估工時**時，第一個報出的數字主宰了後續討論
- 使用者明明覺得不合理，但反駁起來很軟弱——因為思緒已被錨拉走
- 面對複雜估算時，抓到一個「隨機但具體」的數字就開始往那靠

聚焦：**先自訂底線、再看對方開價**。意識到錨的存在就已經削弱一半。

---

### 4. 倖存者偏誤（Survivorship Bias） → `bias/survivorship-bias.md`

**何時載入**：

- 使用者只研究**成功者的做法**（「這些 CEO 的習慣」「這些獨角獸的策略」）
- 看到暢銷書、創業故事就想複製，沒追問「有多少人同樣做法失敗了」
- 評估投資 / 職涯 / 賽道時只看「做這行成功的人」

聚焦：**問「同樣做法失敗的人在哪？」** 看不見的失敗者才是答案。

---

### 5. 鄧寧-克魯格效應（Dunning–Kruger Effect） → `bias/dunning-kruger.md`

**何時載入**：

- 使用者**剛入門某領域**就講得頭頭是道、判斷大膽
- 或相反：在**自己專長**裡反而覺得「我其實不懂」，不敢發言
- 團隊裡新手最敢拍板、專家最猶豫，決策品質反向
- 自媒體 / 社群中，最多按讚的論點常來自不夠深的觀察

聚焦：**新手常過度自信，專家常過度謙虛**。校準來自持續被現實打臉。

---

### 6. 過度自信偏誤（Overconfidence Bias） → `bias/overconfidence.md`

**何時載入**：

- 使用者對**預測、估工時、投資報酬**給出過於精確的數字（「一定」「絕對」「一週就能做完」）
- 決策時沒有區間、沒有信心度，只有點估計
- 事後看過往預測命中率遠低於自己以為的

聚焦：**給區間，不給點**。事後記錄 / 回看預測命中率，才會校準。

---

### 7. 損失規避（Loss Aversion） → `bias/loss-aversion.md`

**何時載入**：

- 使用者因為**怕失去**而錯過明顯更好的機會（不換工作、不停損、不離開壞關係）
- 沉沒成本推著走：「已經投入這麼多」
- 面對「有機會小賺 / 小機率大賠」vs「穩定小賠 / 小機率大賺」時直覺選錯邊

聚焦：**用長期期望值思考**，不用單次得失。損失的痛 ≈ 同樣大小收益的兩倍，要主動校正。

---

### 8. 從眾效應（Bandwagon Effect） → `bias/bandwagon-effect.md`

**何時載入**：

- 「大家都這樣說」「這個超紅」成了使用者的主要理由
- 投資、職涯、消費跟著熱潮走（meme stock、AI 熱、網紅推薦）
- 在會議 / 社群裡不敢提異見——因為多數人看起來同意

聚焦：**分開「多少人相信」與「多少證據支持」**。共識可能是群體智慧，也可能是群體幻覺。

---

### 9. 基本歸因謬誤（Fundamental Attribution Error） → `bias/fundamental-attribution-error.md`

**何時載入**：

- 使用者解釋別人錯誤歸因於**個性 / 人品**（「他就是懶」「他態度差」）
- 解釋自己錯誤歸因於**情境**（「今天剛好很累」「系統出問題」）
- 事故 / 衝突檢討變成「找戰犯」而非「改結構」
- 團隊文化中有明顯「怪人不怪系統」的傾向

聚焦：**對人寬、對己嚴；先看情境、再談個性**。多數行為是結構產物，不是人格缺陷。

---

### 10. 敘事謬誤（Narrative Fallacy） → `bias/narrative-fallacy.md`

**何時載入**：

- 使用者把**隨機事件**講成有因有果的故事（「當年若不是那通電話⋯⋯」）
- 事後諸葛：用現在知道的結果，倒推出「其實當時就看得出來」
- 評估投資 / 產品 / 政策時，被**動人故事**說服，忽略機率與樣本
- 看傳記、創業故事就想照劇本走

聚焦：**承認有些事就是雜訊**。不是每件事都有原因，也不是每個成功都有祕訣。

---

## 挑選提示

- **只看支持證據** → 確認偏誤
- **被近期新聞 / 記憶主宰** → 可得性
- **被第一個數字拉走** → 錨定
- **只研究成功者** → 倖存者
- **新手 / 專家自評失準** → 鄧寧-克魯格
- **預測太肯定** → 過度自信
- **怕失去大於想獲得** → 損失規避
- **跟著熱潮走** → 從眾
- **怪人不怪系統** → 基本歸因
- **為隨機事件編故事** → 敘事謬誤

## 跨 submodule 的常見組合

偏誤很少單獨出現。典型群體：

- **「熱門股跌了我還不賣」**：從眾（跟進）+ 錨定（買入價）+ 損失規避（不敢停損）+ 敘事（說服自己這只是短期回檔）
- **「這個創業我要模仿」**：倖存者（只看成功）+ 確認（找支持自己的故事）+ 敘事（相信「成功必有原因」）+ 過度自信（我也做得到）
- **「團隊又出包了，就是他們不認真」**：基本歸因 + 確認偏誤 + 可得性（想到最糟案例）
- **「新人第一天就想改架構」**：鄧寧-克魯格 + 過度自信
- **「大家都在唱衰經濟，應該沒救了」**：從眾 + 可得性（負面新聞）+ 敘事（把雜訊串成末日故事）

看到偏誤群組時，不要試圖一次拆完——挑最當下、最可操作的那個切入。

---

## 使用偏誤模組的守則

1. **偏誤是診斷工具，不是罵人工具**。指出別人有偏誤前，先問自己有沒有同一個。所有模型也適用「自己先用」。
2. **知道偏誤不等於避開偏誤**。讀過《快思慢想》的人一樣會被錨定。對策是**流程**（事前寫下判準、事後檢討命中率），不是**意志**。
3. **偏誤不是全錯**。多數偏誤在演化上是**有用的捷思**——速度換準度。關鍵是在**高風險、慢思考才需要的場合**把它關掉。
4. **不要過度歸咎偏誤**。有時人的結論就是對的，只是和你不同。把對手打成「你有 XX 偏誤」是思辨的偷懶。
