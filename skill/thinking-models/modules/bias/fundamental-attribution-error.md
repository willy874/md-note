# 基本歸因謬誤（Fundamental Attribution Error）

## 核心概念

基本歸因謬誤由 Lee Ross 在 1977 年提出：**解釋別人的行為時，傾向歸因於個性（dispositional）；解釋自己或自己人的行為時，傾向歸因於情境（situational）**。

典型對照：

- 別人開車很急 → 「他是個自私的人」
- 自己開車很急 → 「我今天要趕去接小孩」
- 別人遲到 → 「他沒有時間觀念」
- 自己遲到 → 「剛剛有個緊急狀況」
- 同事績效差 → 「他能力不行」
- 自己績效差 → 「今年景氣 / 團隊支援不足 / 需求一直改」

這個不對稱之所以會發生，是因為**視角不同**：
- 看自己時，你看到的是**情境**（我眼前的擁塞、我今天的壓力），自己的性格是不可見背景
- 看別人時，你看到的是**人**（對方的行為、表情、選擇），情境是不可見背景

結果是**對人嚴、對己寬**，而且多半不自覺。組織層面衍生出「怪人不怪系統」的文化——每次出事都找戰犯，不改流程。

核心洞見：**多數行為是情境產物，不是人格缺陷**。人在不同情境下會做出截然不同的選擇。把情境因素忽略掉，等於把大部分解釋力扔掉。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者解釋**別人的錯誤 / 失誤** 時歸因於人品、能力、態度（「他就是懶」「她態度差」「那個人就這樣」）
- 解釋**自己的失誤**時歸因於情境（「今天剛好很累」「那天系統出問題」「客戶太難搞」）
- **事故 / 衝突檢討**變成「找戰犯」而非「改結構 / 改流程」
- 團隊文化裡明顯有「怪人不怪系統」的傾向，重複出錯但沒人改根因
- 使用者對**陌生人、外團體**（不同部門、不同國家、不同立場）下強烈的人格判斷
- 人際關係中的**長期誤解** 來自一次情境表現被貼成永久標籤

不適合時機：

- 有**穩定的行為模式證據**（長期多情境都如此）——那個判斷可能就是對的
- **個人責任**真的是問題核心時——過度歸因情境也會變成幫人找藉口

## 如何應用

1. **對人先問情境**：看到別人的負面行為，先問「他可能正處於什麼情境？」而不是「他是個怎樣的人？」。通常三個合理的情境猜想就足以讓你暫緩下人格判斷。
2. **對自己先問責任**：遇到自己的失誤時，反過來——先問「如果我是個旁觀者，他們會怎麼看我這個行為？」。這叫**攝影機視角**。
3. **用「如果是我，在同樣情境下會怎樣？」取代「他怎麼可以這樣？」**：同理心的第一步。多數時候你會發現「在那個情境下我可能也會這樣」。
4. **事故檢討用「[5 Whys](../analysis/five-whys.md)」而不是「誰做的」**：把焦點從「追究個人責任」轉到「流程在哪一環容許這件事發生」。Toyota 的精神：對事不對人。
5. **建立多樣本**：對某人的判斷至少要三個不同情境的觀察。一次情境的樣本容易把情境誤當作人格。
6. **對組織文化的影響**：作為領導 / 管理者，每次事故如果只處罰個人不改流程，會讓下次同類事件繼續發生，且團隊開始隱瞞問題。
7. **警覺「外團體」的強化版**：我們對自己人比較容易給情境解釋，對外團體（不同部門、不同政黨、不同國家）最常直接人格歸因。重要判斷前問自己：「如果這件事是我們部門做的，我會怎麼解釋？」

## 案例

- **Jones & Harris 實驗（1967）**：給受試者讀支持 / 反對卡斯楚的文章，明確告知「作者是被指派立場、不是自願」。受試者**仍然**認為作者本人傾向那個立場。情境（被指派）明擺著，人格歸因仍然主導。
- **公路暴怒**：別人切進你車道 → 「沒品」。你切進別人車道 → 「我這是合理變道」。同樣行為，不同歸因。
- **工作事故**：工程師推了一個壞 commit 導致當機 → 老闆說「他不夠小心」→ 下次換個人仍會發生。真正原因可能是：沒有 staging、沒有 CI、review 流程有漏洞。**人只是暴露了結構問題的觸發器**。
- **遲到**：同事遲到 → 你覺得他沒責任感。你遲到 → 你覺得「今天路況特別糟」。實際上交通真的會變，你和同事的情境資訊是不對稱的。
- **績效評估**：主管看下屬績效差直接歸因能力不足，忽略了「需求改了三次、資源被砍、同事離職導致工作翻倍」這些情境因子。造成優秀人才被誤判流失。
- **跨部門衝突**：業務覺得工程「不配合」、工程覺得業務「不懂專業」。雙方都用人格 / 能力解釋對方，忽略了**雙方 KPI 結構把他們逼到對立**。
- **跨國 / 跨文化**：「那個國家的人都怎樣怎樣」——把一個巨大情境（歷史、制度、經濟結構）打包成民族性格。強烈的基本歸因謬誤。
- **對陌生人短暫觀察**：便利商店店員態度差 → 你覺得他「工作不用心」。可能他剛被客訴、剛被主管罵、家裡出事。你只看到 30 秒，卻下了人格結論。

## 常見誤用

- **極端反轉變成「都是社會的錯」**：發現基本歸因謬誤後，走向「一切都是情境，個人沒責任」。另一種極端。人格和情境都是因素，權重要根據證據。
- **用「情境解釋」幫壞行為脫罪**：有人長期惡行 → 一句「他也是環境受害者」把責任抹掉。同理不等於縱容。
- **只在被指責時想起「基本歸因謬誤」**：自己被罵就搬出「你們沒看見情境」；指責別人時忘得一乾二淨。只單向使用是偏誤 × 2。
- **把「系統問題」當萬用擋箭牌**：每次個人失職都推給「系統有問題」，組織的改善動能反而被稀釋。系統和個人責任要並行追究。
- **對自己人過度情境歸因**：對自己團隊的錯誤太多情境解釋，導致真正有問題的人沒被校正。基本歸因謬誤的「內團體版本」。

## 與其他模型的關係

- **+ [確認偏誤](confirmation-bias.md)**：一旦對某人貼上標籤，後續所有行為都被解讀成「果然如此」。歸因偏誤 → 標籤 → 確認偏誤 → 更深的標籤。
- **+ [可得性偏誤](availability-heuristic.md)**：最近看到的一兩個行為樣本被當成「這個人就是這樣」的全部證據。
- **+ [從眾效應](bandwagon-effect.md)**：團隊中對某人的負面共識會相互放大——大家都說他不行 → 你也不再獨立觀察。
- **+ [系統思考](../analysis/systems-thinking.md)**（[analysis 模組](../analysis/index.md)）：基本歸因謬誤的解毒劑之一。學會看結構、看流程、看誘因，就不會一直怪個人。
- **+ [5 Whys](../analysis/five-whys.md)**（[analysis 模組](../analysis/index.md)）：事故檢討時 5 Whys 幫你穿透「是誰做的」追到「流程為什麼容許」。
- **+ [魚骨圖](../analysis/ishikawa.md)**（[analysis 模組](../analysis/index.md)）：魚骨圖的類別（人 / 機 / 料 / 法 / 環）天生把「人」和「環境」並列展開——提醒自己不要只盯「人」那根骨。
- **+ [learning 模組](../learning/index.md)的[類比](../learning/analogy.md)**：跨文化、跨部門溝通時，先把「如果我是他們、在他們的情境」想清楚再評論。
