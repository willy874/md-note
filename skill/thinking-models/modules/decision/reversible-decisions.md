# 可逆 vs 不可逆決策（Two-Way vs One-Way Doors）

## 核心概念

Jeff Bezos 在 1997 年 Amazon 股東信提出這個比喻：

- **雙向門（Two-way door）**：進去後發現錯了可以退回。錯的成本低，可以快速嘗試、快速修正。
- **單向門（One-way door）**：一進去就出不來。錯的成本極高，必須謹慎、收集更多資訊、多人決策。

Bezos 的關鍵觀察：**多數決策是雙向門，但被錯當成單向門處理**——過度謹慎、層層簽核、決策癱瘓。同時，**少數決策是單向門，卻被錯當成雙向門處理**——「試試看吧」，結果造成不可逆的災難。

兩種錯配都會導致問題：

1. **把雙向門當單向門**：速度慢、錯失機會、養成過度謹慎的組織文化。Amazon 不想變成這種大公司。
2. **把單向門當雙向門**：頭鐵衝、燒橋、公開失言、破產 —— 救不回來。

決策的正確節奏：**雙向門要快，單向門要慢**。兩種速度錯配，就是浪費或災難。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者在拿捏**要多謹慎、要不要再想一下**
- 團隊決策節奏：瓶頸 vs 敏捷
- 重大人事（雇用、解雇）、併購、公開承諾、資本配置
- 「我怕做錯」導致所有決定都過度謹慎——或反之，一路衝沒煞車
- 新創 vs 大公司的決策風格比較
- 個人生活：辭職、搬家、結婚、生小孩、大額消費

不適合時機：

- 純執行性任務（照 SOP 做即可）
- 可逆性已經非常明確的情境——不需要分析框架
- 決策本身不重要（低 stakes）

## 如何應用

1. **第一個問題永遠是**：這是單向門還是雙向門？寫下來，不要用感覺判斷。
2. **判斷指標**：
   - 能不能取消 / 退貨 / 更正？
   - 取消的成本（錢、時間、關係、聲譽）有多高？
   - 時間窗口：有多久可以改？
3. **雙向門決策**：快速做、邊做邊學，錯了退回。不要過度分析。
4. **單向門決策**：慢下來、收集證據、多角度討論、做 Pre-mortem、寫 worst-case。
5. **把單向門切成多個雙向門**（MVP 思維）：不能直接 all-in，就分階段驗證——試點、小規模、可撤回的承諾。
6. **為單向門建保險**：合約中的 exit clause、緩衝期、revert 機制、回購條款。
7. **事後回顧**：上次的決策到底是不是單向門？判斷對了嗎？

## 案例

- **發布新功能**：通常是雙向門——用戶不喜歡就下線。所以 Amazon、Netflix 快速發佈、快速回收。
- **大規模裁員**：單向門——聲譽、信任、團隊士氣一旦崩塌很難回復。決策前要非常謹慎。
- **公開道歉 / 公開發言**：單向門——話一出口，收不回來。CEO 上 Twitter 發飆、政治人物失言，代價巨大。
- **辭職**：多數人以為是單向門，其實很多情況可以回（前公司可能歡迎你回去）。但某些情況是真的回不去（燒橋、產業黑名單）。要個案判斷。
- **結婚 / 生小孩**：接近單向門。法律上可以離婚，但時間、情感、財務的不可逆性很高。
- **創業燒光存款**：燒完存款才發現不對——單向門。先做 side project 驗證，變成雙向門。
- **簽長約**：原本可能是雙向門（短期雇傭），簽了三年不可解約條款，立刻變單向門。

## 常見誤用

- **把雙向門當單向門**：最常見的大公司病。連改個 UI 顏色都要五層簽核——每個雙向門都被當單向門處理，組織逐漸石化。
- **把單向門當雙向門**：「沒事啦，不行再改回來」——燒橋、公開侮辱客戶、裁員後再請回來。表面上可以「改」，實際上信任已失。
- **誤判可逆性**：有些門**表面可逆，實質不可逆**——例如員工資遣費付了、對方不願意回來；或是「刪除了也可以復原」但資料已經外洩。要誠實評估「退回」的真實成本。
- **一刀切套用速度**：有人聽完 Bezos 就「所有決策都要快」——忽略單向門也在其中。速度要匹配可逆性。
- **用來合理化衝動**：「反正可逆啦」——但可逆不代表零成本，每次錯誤都有學習成本、時間成本、他人信任成本。

## 與其他模型的關係

- **+ MVP**（action 模組）：MVP 是把單向門切成雙向門的核心工具——不要 all-in，先做能驗證假設的最小版本。
- **+ 期望值**：雙向門可以只看平均期望值（反正錯了能退），單向門要看**最壞情況**是否可承受。
- **+ Pre-mortem**：單向門決策前**必做**。雙向門可以省略——反正錯了再改。
- **+ OODA**（action 模組）：雙向門要追求循環速度，OODA 節奏快；單向門要放慢 Decide，Observe 和 Orient 要更徹底。
- **+ 沉沒成本**：單向門決策容易產生 sunk cost 陷阱（都已經進來了，不能退）。事前認清單向門，事後比較不會被 sunk cost 綁架。
- **+ 機會成本**：單向門的機會成本包含「未來所有與 B 相關的選擇權」。估算時要含進去。
