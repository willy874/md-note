# 貝氏更新（Bayesian Updating）

## 核心概念

貝氏定理來自 18 世紀牧師 Thomas Bayes，公式：

> P(H|E) = P(E|H) × P(H) / P(E)

白話翻譯：**「看到新證據後我該相信這個假設幾分」＝「假設為真時看到這個證據的可能性」× 「原本就相信假設為真的程度」 ÷ 「任何情況下看到這個證據的可能性」**。

實務上不需要硬背公式。貝氏更新的精神是：

1. **信念是機率，不是 True/False**：理性的人不是「相信」或「不相信」一件事，而是**給它一個機率**（30%、70%、95%）。
2. **Prior（先驗）+ Evidence（證據）= Posterior（後驗）**：新證據進來時，調整機率，而不是推翻重來或完全無視。
3. **更新幅度要合理**：強證據大幅更新，弱證據小幅更新。被單次事件嚇到 180 度翻盤，和完全無視證據，都是失敗的更新。

Nate Silver、Philip Tetlock（Superforecasters 研究）都發現——長期預測表現最好的人，共同特徵是**頻繁、小幅、誠實地更新信念**，而不是堅守立場或被單次事件嚇傻。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者面對**持續更新的證據流**：市場、選舉、競品動態、實驗數據、財報
- 預測類問題（這個政策會通過嗎？這場會議會成功嗎？）
- 科學思考、debug 推理（「這個 bug 是 X 造成的機率多少？這個 log 訊息改變了多少？」）
- 使用者**死守初始立場**，對反面證據視而不見（確認偏誤）
- 或反之，使用者**被單次新聞嚇到大幅翻盤**（可得性偏誤）
- 閱讀新聞、社群爆料時——資訊過載需要一個更新框架

不適合時機：

- 純價值觀 / 倫理判斷——這些不是機率問題
- 完全沒有先驗、沒有 base rate 的全新領域（這時先做 exploration）
- 使用者情緒在高點——先處理情緒再談理性更新

## 如何應用

1. **寫下你的先驗信念**：用**機率**表達，不用「有點相信」這種模糊詞。例：「這個產品會做起來的機率 30%」。
2. **新證據到來**，問自己兩個關鍵問題：
   - **如果假設為真**，看到這個證據的機率有多大？
   - **如果假設為假**，看到這個證據的機率有多大？
3. **更新幅度**：兩個機率差距大 = 強證據、大幅更新；兩個差不多 = 弱證據、小幅更新。
4. **寫下新機率**：例：「看到第一週數據後，我把機率從 30% 調到 45%。」
5. **持續追蹤**：每次新證據都做一次。累積下來能看到自己信念的軌跡是否合理。
6. **定期回顧**：過去三個月的預測，哪些對、哪些錯？更新速度太慢還是太快？調整自己的更新偏誤。

## 案例

- **醫療診斷**：病人有某症狀，醫生不是直接下診斷。他考慮 base rate（該疾病在人口中的比例）+ 這個症狀在該病 vs 其他病的機率。這就是 Bayesian thinking。忽略 base rate 是醫學誤診常見原因。
- **面試**：履歷 = prior（例：70% 適合）+ 面談 = evidence。面談表現差 → 下修到 40%，不是 0%。面談表現好 → 上修到 85%，不是 100%。一次面試翻盤極少是合理的。
- **市場預測**：某產品預估轉換率 5%（先驗）。第一週實測 3%（證據）。理性更新：大幅下修但不要 panic——因為第一週樣本可能太小，先驗還有一定權重。
- **投資**：對公司的信心隨每季財報更新。巴菲特不會因一季壞就賣，也不會因一季好就加碼——會看**這是噪音還是趨勢**，決定更新幅度。
- **人際判斷**：朋友這次遲到。他是「不尊重別人」嗎？prior（他平常準時）+ evidence（這次遲到的脈絡）→ 小幅下修，而不是貼標籤。

## 常見誤用

- **更新太慢（確認偏誤）**：死守既有信念、對反面證據視而不見。各種「這次不算」「特殊情況」。
- **更新太快（可得性偏誤）**：被單次事件嚇到大幅翻盤——「這家公司一次醜聞 → 完蛋了」，其實可能只是噪音。
- **忽略 base rate**：只看眼前證據，忘記先驗機率。最著名的例子：一個看起來很像律師的人，你猜他是律師還是工程師？多數人直覺答律師——但如果工程師人口是律師的 10 倍，base rate 會壓過相似度。
- **假裝精確**：把感覺 35% 寫成「確信 38.5%」——過度精確反而掩蓋真實不確定性。範圍（30-40%）誠實得多。
- **只對自己反對的證據嚴格**：看到支持自己立場的證據就接受，看到反對的就挑毛病。這是確認偏誤的典型——要對兩邊證據用同樣標準。
- **把更新當投機**：每次新聞就翻盤一次，沒有信念的錨點——變成隨波逐流，不是貝氏更新。

## 與其他模型的關係

- **+ 期望值**：機率估計要隨證據更新，期望值才會準。Bayesian updating 是期望值思考的**動態燃料**。
- **+ OODA**（action 模組）：OODA 的 Orient 本質上就是持續貝氏更新——環境變了，你對局勢的信念要跟著調。
- **+ bias 模組**：**確認偏誤、可得性、錨定、過度自信**都是 Bayesian updating 失敗的典型症狀。做重要更新前要做一次偏誤健檢。
- **+ 二階思考**：新證據可能還有**其他假設**也符合——不要只問「這符合我的假設嗎」，要問「還有什麼假設也符合」。
- **+ Pre-mortem**：Pre-mortem 列出的風險可以變成預警指標——每次有新證據，更新這些風險的機率。
- **vs 直覺決策**：直覺適合高度熟悉、回饋快的領域（Kahneman 的 System 1）。貝氏更新適合不熟悉、回饋慢、需要外顯推理的領域。兩者不對立，而是適用場景不同。
