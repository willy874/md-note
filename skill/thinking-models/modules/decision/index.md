# 二、決策與判斷

> 載入時機見 `SKILL.md` 的路由表。本檔為此類別的**子路由**——先對照下表判斷使用者的情境，再 Read 對應的 submodule 深入取用。

用來在資訊不完整、有風險的情況下做出較好選擇。八個 submodule 對應八種不同的決策情境，不要一次載入全部。

---

## Submodule 子路由表

### 1. 期望值思考（Expected Value） → `decision/expected-value.md`

**何時載入**：

- 使用者面臨**可重複、可量化**的選擇（投資、商業決策、A/B 測試、下注型選項）
- 討論「高機率小贏 vs 低機率大贏」的取捨
- 使用者因**單次結果**懊悔或得意，錯把結果當決策品質
- 機率與 payoff 都能大致估計的場景

聚焦：**決策品質 ≠ 結果品質**。長期看期望值，不看單次。

---

### 2. 反向思考（Inversion） → `decision/inversion.md`

**何時載入**：

- 正向討論「怎樣成功」太發散、沒結論
- **風險管理**、產品設計、人生規劃，需要先排除地雷
- 使用者陷在「完美解法」的追求裡，進度停滯
- 團隊對目標有共識，但對風險缺乏討論

聚焦：**Invert, always invert**。避開失敗往往比追求完美更可行。

---

### 3. 機會成本（Opportunity Cost） → `decision/opportunity-cost.md`

**何時載入**：

- **時間 / 金錢 / 注意力** 的分配取捨
- 職涯選擇、合作邀約、投資選項
- 使用者只算「做這件事的花費」，沒算「放棄了什麼」
- 「我很忙但沒產出」——往往是機會成本被顯性工作吃掉

聚焦：**選 A 的代價是放棄的最佳替代 B**。隱性成本常常比顯性更大。

---

### 4. 沉沒成本謬誤（Sunk Cost Fallacy） → `decision/sunk-cost.md`

**何時載入**：

- **停損**決策：失敗專案、不健康關係、看不到終點的堅持
- 「都已經投入這麼多了，不能放棄」的語氣
- 職涯轉換：「已經做了 X 年⋯⋯」
- 投資加碼「攤平」、賭桌「翻本」思維

聚焦：**已花出去且收不回的，不該影響未來決策**。只看往後的成本與收益。

---

### 5. 二階思考（Second-Order Thinking） → `decision/second-order-thinking.md`

**何時載入**：

- **政策、策略、組織規則**制定——顯而易見的解往往是錯的
- 使用者只看到直接後果，沒想到「然後呢？再然後呢？」
- 投資 / 市場預測（多數人的一階共識已經 priced in）
- 產品功能討論：加了會誘發什麼用戶行為、競爭對手會怎麼反應

聚焦：**一階人人會，差距在二三階**。找出非預期後果與反撲機制。

---

### 6. 可逆 vs 不可逆決策（Two-Way vs One-Way Doors） → `decision/reversible-decisions.md`

**何時載入**：

- 使用者在拿捏**要多謹慎、要不要再想一下**
- 團隊決策節奏：瓶頸 vs 敏捷
- 重大人事、併購、公開承諾、資本配置
- 「我怕做錯」導致所有決定都過度謹慎——或反之，一路衝沒煞車

聚焦：**區分單向門與雙向門**。雙向門快做、單向門慢做，錯配就是浪費或災難。

---

### 7. 最壞情況分析（Pre-mortem） → `decision/pre-mortem.md`

**何時載入**：

- **重大專案啟動前**、不可逆決策前
- 團隊風險討論**氣氛一面倒樂觀**（Groupthink 徵兆）
- 發布、併購、重組、上線前最後一次 sanity check
- 使用者個人的重大人生選擇（讀研所、創業、搬家）

聚焦：**假設它已經失敗了，回推原因**。把批評「計劃中的風險」變成討論「虛擬失敗的原因」。

---

### 8. 貝氏更新（Bayesian Updating） → `decision/bayesian-updating.md`

**何時載入**：

- 使用者面對**持續更新的證據流**：市場、選舉、競品、實驗數據
- 預測類問題、科學思考、debug 推理
- 使用者**死守初始立場**，對反面證據視而不見
- 或反之，**被單次新聞嚇到大幅翻盤**

聚焦：**信念是機率，不是真 / 假**。新證據來了調整幾分，而非推翻重來。

---

## 挑選提示

- **可量化、可重複的選項比較** → 期望值
- **先排除地雷再談成功** → 反向思考
- **取捨 A vs B** → 機會成本
- **該不該停損** → 沉沒成本
- **顯而易見的解可能錯** → 二階思考
- **要多謹慎的拿捏** → 可逆 vs 不可逆
- **重大決策前 sanity check** → Pre-mortem
- **信念該隨證據調多少** → 貝氏更新

多數使用者需求只對應一個 submodule。真的需要跨 submodule 時，典型組合：

- **重大不可逆決策**：可逆性判斷 → Pre-mortem → 二階思考
- **停損困難**：沉沒成本 + 機會成本（兩者同時壓著使用者）
- **投資 / 下注**：期望值 + 貝氏更新 + 二階思考（別人的反應也會 priced in）
- **人生選項**：機會成本 + 反向思考 + Pre-mortem
