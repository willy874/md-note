# 系統思考（Systems Thinking）

## 核心概念

系統思考由 MIT 的 **Jay Forrester** 開創（System Dynamics），經 **Donella Meadows**（《Thinking in Systems》）、**Peter Senge**（《第五項修練》）普及。核心主張：

> 「世界不是由彼此獨立的事件組成，而是由**互相連結的關係**組成。問題看起來是 A 引起 B，其實常常是 B 也引起 A。」

傳統的線性因果（A → B）在簡單問題上好用，但面對組織、市場、生態、政策時會失效。系統思考的三個核心元素：

1. **回饋循環（Feedback Loops）**：
   - **增強迴路（R）**：A 增加 B，B 又增加 A。例如「口碑 → 用戶增加 → 口碑更強」（滾雪球），但壞事也會滾（恐慌 → 拋售 → 更恐慌）。
   - **調節迴路（B）**：A 增加 B，B 反過來抑制 A。例如「冷氣越開越冷 → 到設定溫度就停 → 溫度回升再開」（恆溫器）。

2. **延遲（Delay）**：因與果之間有時間差。沒意識到延遲會導致過度反應（例：洗澡水太冷 → 猛轉熱 → 幾秒後燙到）、也會讓問題累積多年才爆發（例：慢性病、組織文化腐蝕）。

3. **槓桿點（Leverage Points）**：Donella Meadows 列出 12 種，從低到高是：參數 < 規則 < 資訊流 < 誘因結構 < 典範。**改典範的威力遠大於改數字**——但多數人只敢動參數。

關鍵洞見：**多數「越解決越糟」的現象，是因為我們只看到線性片段，沒看到系統結構**。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- 使用者描述「**越努力越糟 / 治標不治本 / 按下葫蘆浮起瓢**」——典型的系統反撲
- 問題涉及**多方互動、長時間尺度、意料之外的副作用**（組織、政策、市場、生態、公共衛生）
- 單點優化讓**局部變好但整體變糟**（local max vs global max）
- 要找**槓桿點**：想用小改變引發大改變
- 討論**結構性問題**（貧富差距、壅塞、氣候、組織文化、技術債）

不適合時機：

- 簡單線性問題（機器壞了、bug 能追到一行 code）→ 5 Whys 或魚骨圖
- 急需決策的情境 → 系統思考很慢，OODA / decision 模組更合用
- 使用者只想要「具體怎麼做」——系統思考產出「結構洞察」，不是清單

## 如何應用

1. **畫出變數之間的關係圖**：
   - 列出關鍵變數（不只看眼前問題，也看上下游）
   - 用箭頭標出「誰影響誰」，標上 `+`（同向）或 `-`（反向）
   - 找出**回饋循環**（箭頭繞回原點）
2. **辨識是增強還是調節**：
   - 增強迴路 → 滾雪球（好壞都會放大）
   - 調節迴路 → 維持穩定（但也會抗拒改變）
3. **找「延遲」**：哪裡的「因 → 果」有時間差？延遲越長，越容易過度反應或長期忽略。
4. **找槓桿點**：想干預系統時，問「我能動哪一層？」——越往上層（規則、誘因、典範）威力越大，但阻力也越大。
5. **警覺「系統反模式」**（Senge 歸納）：
   - **治標不治本（Shifting the Burden）**：短期症狀方案讓根因惡化（吃止痛藥 → 沒治肝）
   - **成長極限（Limits to Growth）**：某個增強迴路被調節迴路掐住（用戶暴增 → 系統撐不住 → 體驗變差）
   - **悲劇共享（Tragedy of the Commons）**：每人短期最優 → 整體崩潰（漁場、停車位、共用會議室）
6. **做「低後悔度介入」**：系統很難預測，優先選「即使錯了也能回復」的干預。

## 案例

- **塞車越蓋路越堵**（Braess's Paradox + 誘發需求）：蓋新路 → 開車變便利 → 更多人改開車 → 回到原狀或更糟。槓桿點不在「蓋路」，在「通勤誘因結構」（大眾運輸、遠距工作、擁擠費）。
- **越減薪越燒錢**：公司業績差 → 減薪 → 好員工先離職 → 剩下的士氣低 → 業績更差。增強迴路惡性循環。槓桿點是停止減薪 + 投資核心員工。
- **組織技術債**：趕交付 → 寫 shortcut → debt 累積 → 新功能更慢 → 更趕 → 更多 shortcut。延遲讓這種迴路在前兩季看不出來，第三季開始雪崩。
- **慢性病 / 健康**：壓力 → 睡眠差 → 隔天更累 → 決策差 → 更多壓力。多數生活危機是增強迴路 + 延遲。槓桿點是早期睡眠介入，而不是危機發生後補救。
- **產品 viral loop**：推薦碼 → 新用戶 → 更多推薦 → 更多新用戶。設計成功的 growth engine 本質上是刻意打造的增強迴路。
- **通膨預期**：預期物價上漲 → 提前買 / 加薪要求 → 真的漲 → 預期更強。央行的工作多半是打斷這個增強迴路。

## 常見誤用

- **畫很複雜的圖但沒行動**：系統圖畫到五十個節點、二十條回饋——看起來很深，但沒辦法改任何事。Meadows 強調：**簡化到能動手的最少節點**。
- **只看結構不看證據**：憑直覺畫關係，沒有資料支撐。系統圖是「思考工具」，不是「事實」。
- **誤把簡單問題當系統問題**：一個壞掉的螺絲不需要系統思考，直接換掉就好。殺雞用牛刀。
- **找到「典範層級的槓桿點」就想直接改**：典範威力最大但阻力最大。不切實際地鎖定它，常常一事無成。實務上很多改變從「資訊流」「規則」就能見效。
- **忽略延遲就評估干預**：改完政策看兩週沒效果就放棄，其實效果在一年後才出現。
- **把系統思考講得神秘兮兮**：「這是複雜系統的湧現性⋯⋯」——聽起來高深但沒產生任何可操作的洞見。好的系統思考是**把複雜講簡單**，不是反過來。

## 與其他模型的關係

- **vs 5 Whys / 魚骨圖**：兩者假設「線性因果」——找到原因就能修。系統思考面對的是**非線性 + 回饋**的情境，需要不同工具。實務上先用 5 Whys / 魚骨圖，若發現回饋循環再切換系統思考。
- **+ 第一性原理**：系統圖會暴露哪些「慣性假設」其實建立在某個迴路之上，挑戰假設的機會就在那裡。
- **+ decision 模組的二階 / 三階思考**：兩者精神高度重合——都強調「這個行動之後，世界會如何反應」。二階思考是系統思考的短版。
- **+ compounding**：增強迴路就是複利的系統版本。
- **+ bias 模組**：系統思考最常被**短視偏誤**（只看眼前）、**基本歸因謬誤**（怪人不怪結構）劫持。畫完圖要回頭檢查。
- **+ action 模組 MVP**：系統介入風險高，先做小範圍 MVP 驗證再推廣。
