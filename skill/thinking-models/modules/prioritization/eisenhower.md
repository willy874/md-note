# 艾森豪矩陣（Eisenhower Matrix）

## 核心概念

美國總統 Dwight Eisenhower 在一次演講中引述某大學校長的話：
> 「我有兩種問題——緊急的和重要的。緊急的不重要，重要的從來不緊急。」

後來被 Stephen Covey 在《高效能人士的七個習慣》整理成一個 2×2 矩陣：

|          | 緊急                       | 不緊急                         |
|----------|----------------------------|--------------------------------|
| **重要** | Q1：危機、死線（**Do**）   | Q2：規劃、學習、健康（**Plan**） |
| **不重要** | Q3：干擾、救火（**Delegate**） | Q4：雜訊、社交媒體（**Delete**）   |

核心洞察：**多數人卡在 Q1 + Q3**（整天救火）、**靠 Q4 放鬆**（滑手機逃避），而**真正決定長期差距的 Q2 永遠沒時間做**。

- **Q1**：已經火燒屁股了，當然要做。但如果 Q1 一直在發生，代表你的 Q2 沒做好（預防、系統性解決）。
- **Q2**：**長期複利的發生地**。健身、學習、寫作、關係、策略規劃——這些不做也不會立刻死，但決定 5 年後你在哪。
- **Q3**：最陰險的象限。別人的急事看起來像你的急事，其實可以委派、拒絕、延後。
- **Q4**：純粹的時間黑洞，用來逃避壓力。真正放鬆不算 Q4——那應該放在 Q2。

## 使用時機

當使用者處於以下情境，載入此 submodule：

- **每天都在救火**、覺得很忙卻沒成長
- 待辦爆炸，分不出「要先做什麼」
- 「重要但不緊急」的事（運動、寫作、學習、策略、健檢）永遠被擠掉
- 容易答應別人的請求，自己的事一直延後
- **個人工作管理、會議安排、訊息回覆分流**
- 使用者描述「一整天過得很充實但說不出做了什麼」

不適合時機：
- 極端危機模式（公司要倒了、家人住院）——這時候全部都是 Q1，不需要分類
- 高度創意 / 探索階段——硬把探索塞進四象限會扼殺可能性
- 已經分類得很清楚、只是執行上卡關——用 timeboxing 或兩分鐘法則（action 模組）

## 如何應用

1. **列出當前所有事項**：週計畫或一天一次，全部倒出來寫。
2. **各自判斷「重要」和「緊急」**：
   - **重要**：和你的**長期目標 / 核心角色**有關（North Star 的延伸）
   - **緊急**：有明確死線、不做會出事
3. **放進對應象限**：
4. **按象限處理**：
   - Q1：**Do**——立刻做。但做完要檢討：為什麼變成 Q1？是否可以預防？
   - Q2：**Plan**——排進行事曆保留時段（不是「有空再做」）
   - Q3：**Delegate**——委派、拒絕、或協商延後。別人的急事不該 default 優先於你的重要
   - Q4：**Delete**——砍掉、至少限制時間
5. **每週檢討象限分佈**：理想上 Q2 應該佔 60-70%，實際多數人不到 20%。
6. **預約 Q2 時段**：運動、學習、寫作、深度工作——預先排進日曆，把它當成老闆約的會議，不可取消。

## 案例

- **工作**：收到同事「幫忙查一下這個數字」的訊息。看起來緊急，但不是你的重要。→ Q3，委派或延後一小時批次處理。
- **副業**：想寫部落格但「最近太忙」。寫作 = Q2。如果不主動排時間，永遠會被 Q1 + Q3 擠掉。→ 週三晚上 7-9 點固定寫作時段，不可取消。
- **健康**：健檢、運動、睡眠充足。典型 Q2，不做不會立刻死，但 10 年後差距巨大。→ 排進行事曆。
- **訊息處理**：同事的問題是 Q3，主管的策略提問是 Q1 或 Q2。別讓通訊軟體的「即時性」騙你所有訊息都一樣重要。
- **家庭時間**：跟家人吃飯、陪小孩、跟伴侶認真聊天——都是 Q2。不排就會一直拖，等到關係出問題就是 Q1。

## 常見誤用

- **所有事都標重要**：人性會把什麼都看成重要。強制篩選——如果所有都重要，等於沒排優先序。
- **混淆「緊急」和「重要」**：Slack 訊息 = 緊急不重要（Q3）、長期學習 = 重要不緊急（Q2）。多數人把 Q3 當 Q1 處理。
- **Q4 當休息**：滑短影片、亂看新聞不是真正的放鬆，是逃避。真正的 recharge 屬於 Q2（運動、睡眠、有質感的嗜好）。
- **分類完沒行動**：分類本身沒價值，配上「Do / Plan / Delegate / Delete」的動作才有。
- **Q1 常態化**：天天有 Q1 不是高效能，是系統性失敗——代表 Q2 的「預防工作」沒做。要問：為什麼這個 Q1 一再發生？
- **誤用在團隊**：把矩陣拿去要求別人「這不重要你不要來吵我」，會毀掉關係。矩陣是**自己內部**的優先排序工具。

## 與其他模型的關係

- **+ 80/20**：80/20 找出「哪些事是你的 20%」，艾森豪矩陣進一步分「現在做還是排進行事曆」。
- **+ North Star**：North Star 定義「重要」的基準，否則每個人對「重要」的定義飄來飄去。
- **+ Impact vs Effort**：Impact vs Effort 用在專案 / roadmap 排序，艾森豪矩陣用在**每日任務**層級。
- **+ 兩分鐘法則**（action 模組）：兩分鐘法則處理 Q3 的小事（快速清掉），艾森豪矩陣處理更大的時間分配。
- **+ 時間盒**（action 模組）：排出 Q2 之後，用 timeboxing 保護那段時間不被侵蝕。
- **vs 完美主義**：完美主義讓你卡在 Q1（出錯恐慌）和 Q4（逃避），矩陣提醒你其實 Q2 才是解。
