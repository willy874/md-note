# Gmail 郵件管理 Skill

## 概述

依據使用者的郵件行為模式，自動判斷郵件應「保留」或「刪除」，並將分類結果整理成表格供使用者手動操作。目前 Gmail MCP 工具僅支援搜尋與讀取，不支援刪除或移動郵件。

## 工具依賴

- `mcp__claude_ai_Gmail__gmail_search_messages` — 搜尋郵件
- `mcp__claude_ai_Gmail__gmail_read_message` — 讀取郵件內容
- `mcp__claude_ai_Gmail__gmail_list_labels` — 列出標籤

## 核心決策規則

### 保留（留在收件夾）

| 類型 | 來源範例 | 原因 |
|---|---|---|
| 銀行帳單 / 對帳單 | 台新、合庫、永豐、北富銀、LINE Bank、國泰世華、新光 | 財務備查 |
| 加密貨幣交易通知 | BitoPro、Kraken | 鏈上操作唯一憑證 |
| 訂閱服務收據 / 電子發票 | Figma、Obsidian、GitHub、ECPay、ezPay | 退款/報帳/稅務憑證 |
| Zoom 會議摘要 | 竹子的個人會議室 | 諮詢紀錄與跟進依據 |
| 有共鳴的 Newsletter | 大大帶我飛、CMoney C Talk+、天下雜誌、Taony Design、Claude Code 更新 | 長效參考價值 |
| 政府 / 健保通知 | 財政部電子發票、中央健保署 | 官方法規文件 |
| 線上課程 / 學習購買確認 | Mina 吳允婷、Readmoo | 完整購買流程脈絡 |
| 生活服務預約（未到期） | 餐廳預訂確認、Starbucks 金星回饋（有兌換資格） | 短期內仍有效 |
| 求職 / 面試相關 | 履歷回覆、面試邀請 | 有實質後續動作 |

### 刪除

| 類型 | 來源範例 | 原因 |
|---|---|---|
| 即時性操作通知 | 銀行 App 登入通知、轉帳完成、消費彙整 | 用完即棄，當下核對後無用 |
| 行銷促銷 / 銀行推銷 | 銀行保險/信貸廣告、Uber Eats/foodpanda 折扣、屈臣氏、寶雅 EDM | 無立即價值 |
| 已過時的提醒 | 已結束活動的訂位提醒、過期折價券提醒 | 時效已過 |
| Uber 實體行程收據 | Uber 車資、外送費用 | 當場核對即結束 |
| 無共鳴的 Newsletter | Frontend Nation、HiSKIO、六角學院職缺 | 未產生共鳴 |
| 社群媒體通知 | LinkedIn、GitHub、X 通知信 | 雜訊 |

## 關鍵決策變數

### 變數一：實體 vs. 網路消費
- **實體消費**（Uber 車資、餐廳、超市）→ 刪除：當場核對即結束，無後續追溯需求
- **網路消費**（訂閱、課程、加密貨幣）→ 保留：有退款/帳號/報帳/稅務等後續風險期

### 變數二：能否事後追溯
- 能從其他管道查到 → 可刪
- 此郵件為唯一憑證 → 必留

### 總結原則
> 「過了就沒用的通知刪，有憑證價值或知識沉澱的留。」

## 執行流程

1. **搜尋**：使用 Gmail 搜尋工具取得待整理的郵件
2. **分類**：逐封依照上述決策規則判斷保留或刪除
3. **呈現**：將分類結果整理成「建議刪除」與「建議保留」兩張表格
4. **提供搜尋條件**：附上可直接貼入 Gmail 搜尋欄的查詢語法，方便使用者快速找到待刪郵件並手動刪除

## 注意事項
- 遇到無法明確歸類的郵件，一律歸入保留並詢問使用者
- 保留郵件的判斷寧可偏寬鬆，避免誤刪有價值的郵件
- 目前無法透過工具直接刪除或移動郵件，僅能提供分類建議與搜尋條件
