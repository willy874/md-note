# 📊 Gmail 郵件行為分析報告
> 分析日期：2026-03-15
> 資料來源：收件夾（已讀未刪）+ 垃圾桶（近期刪除）

---

## 一、收件夾中「不刪除」的郵件類型

### 1. 💰 銀行帳單 & 對帳單（佔比最高）
來源：台新銀行、合作金庫、永豐銀行、台北富邦、LINE Bank、國泰世華、新光銀行

- 每月固定保留完整月對帳單與信用卡帳單
- 視為財務記錄的備查來源

### 2. 🪙 加密貨幣交易通知
來源：BitoPro（USDT/TRX 加值、提領確認）、Kraken（存款/取款）

- 每筆交易通知全部保留
- 原因：鏈上操作的唯一紙本憑證，爭議時無可替代

### 3. 📋 訂閱服務收據 & 電子發票
來源：Figma、Obsidian、GitHub、ECPay、ezPay

- 屬於「網路消費」類別，全數保留
- 退款、帳號異常、報帳、稅務需要時的唯一憑證

### 4. 🎤 Zoom 會議摘要
來源：竹子的個人會議室（技術指導 / 職涯諮詢）

- 保留作為諮詢內容的紀錄與後續跟進依據

### 5. 📰 職涯 & 產業 Newsletter（有共鳴的）
來源：大大帶我飛、CMoney C Talk+、天下雜誌、Taony Design、Claude Code 更新

- 有長效參考價值，可能未來翻閱

### 6. 🏛️ 政府 / 健保通知
來源：財政部電子發票中獎通知、中央健保署、健保電子繳款單

- 官方法規性文件，保留是基本

### 7. 🎓 線上課程 & 學習購買確認
來源：Mina 吳允婷《5週職場翻盤》、Readmoo 電子書家庭帳號邀請

- 保留完整購買流程脈絡（驗證 → 付款 → 歡迎信）

### 8. 🍽️ 生活服務預約 & 有效優惠
來源：Nonsense Bar 餐廳預訂確認、Starbucks 金星會員回饋（有兌換資格）

- 短期內仍有效或需核對的內容

### 9. 💼 求職 & 面試相關
來源：前端工程師履歷回覆（期天資訊）

- 有實質後續動作，加星號保留

## 二、垃圾桶中「選擇刪除」的郵件類型

### 1. ⚡ 即時性操作通知（用完即棄）
- 銀行 App 登入通知（國泰世華 CUBE App、LINE Bank、永豐金證券）
- 轉帳完成通知
- 消費彙整通知（當下核對後即無用）

### 2. 📣 行銷促銷 & 銀行推銷（幾乎全刪）
- 銀行保險 / 信貸廣告
- Uber Eats / foodpanda 折扣序號
- 屈臣氏、寶雅、Starbucks 非回饋類行銷信
- 各品牌促銷 EDM

### 3. ⏰ 已過時的提醒通知
- 訂位提醒（活動結束後刪）
- 折價券到期提醒（POYA）

### 4. 🚗 Uber 實體行程收據
- 單次車資或外送費用
- 當場核對金額後即結束，無後續追溯需要

### 5. 📬 一般 Newsletter（無共鳴的）
- 天下雜誌訂戶週報（部分）
- Frontend Nation、HiSKIO、六角學院職缺
- LinkedIn / GitHub / X 社群通知
- 旅遊推薦、工具行銷更新

## 四、核心決策原則

### 🔑 關鍵變數一：實體 vs. 網路消費
| 類型 | 行為 | 原因 |
|---|---|---|
| **實體消費**（Uber 車資、餐廳、超市）| ✅ 刪 | 當場核對即結束，無後續追溯需求 |
| **網路消費**（訂閱、課程、加密貨幣）| ✅ 留 | 有退款/帳號/報帳/稅務等後續風險期 |

### 🔑 關鍵變數二：「能不能事後追溯」
> **能不能事後追溯**，決定留不留。
>
> 實體消費當場結清，網路消費有後續風險期，所以留著作為唯一憑證。

### 🔑 整體模式

| 共同點 | 說明 |
|---|---|
| 有憑證價值 | 帳單、收據、發票、交易通知 |
| 需要備查 | 銀行記錄、政府文件、交易哈希 |
| 有後續動作 | 面試邀請、預約確認、課程入學 |
| 長效參考價值 | Newsletter、Zoom 會議摘要、技術文章 |
| 情感或紀念性 | 演唱會提醒、重要社群邀請 |

> 📌 **總結**
>
> 「過了就沒用的通知刪，有憑證價值或知識沉澱的留。」
> 你的 inbox 本質上是一個**個人財務 + 職涯 + 學習的知識庫**，而非傳統意義的收件夾。
