# 總會

# 有地方

# 搞得

# 很複雜

#

我向來設計的風格是，我寧可底層邏輯很複雜，我也要業務層看起來簡潔好用。我不願意為了底層少了幾行讓業務層看起來更加複雜或難懂，業務層越好用越好。原因很簡單，因為業務層經常被改動，所以更要簡潔。

當今所有函式庫的原始碼大概 80% 都在服務 API 介面，核心功能其實佔比極少，為得就是讓使用者能舒服的使用。我不能認同「為了讓元件更好維護，業務層出現幾行多餘的宣告是必要之惡」，難道包成開源套件才有資格複雜嗎？

如果大家有想法可以跟我分享唷！
