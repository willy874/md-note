曾經聽過，「CSS 是前端工程師的基本」，但真的是這樣嗎？

應該沒什麼人敢說「我精通 CSS」，這是一門博大精深的語法。

然而現實是，一個 CSS 很強的工程師並不是一個多有商業價值的技能。甚至在面試時，幾乎沒有面試官能問出很有鑑別度的 CSS 面試題。

大多時候，如果單獨是一個語法問題，那麼「查一下」就可以解決。但如果牽扯複雜的佈局條件，他組合可以說千變萬化，在特殊組合下會有意想不到的效果。所以多數情況的問題在於「知道不知道」，而不是單純的「熟練度」。

你有遇過可以堪稱具鑒別度的CSS題目嗎？

希望你能與我分享

- 滾動吧難題
- 過度 0 到 auto
- 三角形
- Stretched link
- 16:9 的 video or iframe
- 標準 Layout
