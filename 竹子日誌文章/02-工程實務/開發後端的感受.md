# 開發後端的

# 感受

我是一個實打實的前端工程師，過去幾年90%都在前端相關領域努力，近期玩了一下後端的開發有一些體感。

- 要更在意事件併發問題，因為不保證所有進入點都是單一執行處理
- 非同步併發整合不多，大部分都可以單線完成處理
- 記憶體不會輕易被清除，要習慣性把多餘的快取清除。
- 進入點很多，但外部依賴相對單純。一個進入點的 Context 單純，不會像前端的外部耦合嚴重。
- 測試超好寫，固定的介面 Mock 之後，大部分複製貼上就可以滿足60%情景。
- 依賴的服務很複雜，要建置一個基礎的開發環境特別費功夫。但環境隔離反而容易，對外接口單純。
- 更在乎抽象介面，而非實作行為。
- 有資料庫服務，容易進行儲存狀態確認與測試。

不知道接下來還可以體驗到什麼，過去都是粗暴的滿足需求寫後端，這次認認真真把功能細化處理。
