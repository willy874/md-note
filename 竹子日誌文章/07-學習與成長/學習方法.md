# 學習

# 方法

#

不少人問我

我怎麼待前端五年就卷到現在這樣

或許我的方法不一定最好

但希望分享給你，讓你有收穫！

1. 下班持續學習：下班後人家懶人家廢，我回家逮到空檔就開始學習。我最高強度時，扣除上班一個月付出上百小時在學習。
2. 上班做有價值的事：工作大部分都在工作，但多多少少有閒的時候，不要浪費時間，做一些提升團隊工作效率的事。寫一些工具、串一些功能、重構一些程式、補一些測試。
3. 除了輸入也要輸出：工程師每個人都拼命學習，但看了就忘記。學習後不管是你要自己寫 SideProject、寫部落格、跟朋友分享，這都是輸出的一種方式。
4. 先深再廣：技術推陳出新，不用一直追。解決方案先學一種，然後瘋狂學深，挖原始碼、理解原作者應用的技巧，接者嘗試模仿去體驗作者解決的問題。等到讀懂了，你會發現同類型的工具理解的很快。
5. 不要逃避：專案中，你一定會遇到「好像不是很懂」的部分，那麼多程式碼和工具，不懂就看，不了解就挖，不要放過眼前的任何可以學習的機會。包含⋯⋯你所有用的工具背後所有的API和原始碼。
